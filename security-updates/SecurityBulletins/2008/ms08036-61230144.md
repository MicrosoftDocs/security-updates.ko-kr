Microsoft 보안 공지 MS08-036 - 중요
===================================

PGM(Pragmatic General Multicast)의 취약점으로 인한 서비스 거부 문제점(950762)
-----------------------------------------------------------------------------

게시된 날짜: 2008년 6월 11일 수요일 | 업데이트된 날짜: 2010년 2월 18일 목요일

**버전:** 1.1

### 일반 정보

#### 요약

이 보안 업데이트는 PGM(Pragmatic General Multicast) 프로토콜에서 발견되어 비공개적으로 보고된 취약점 2건을 해결합니다. 영향을 받는 시스템에서 조작된 PGM 패킷을 받으면 이 취약점을 통해 서비스 거부가 허용될 수 있습니다. 이 취약점을 악용한 공격자는 사용자 시스템이 응답을 하지 못하거나 기능을 복원하기 위하여 다시 시작하도록 만들 수 있습니다. 서비스 거부 취약점으로 인해 공격자가 코드를 실행하거나 사용자 권한이 상승되는 것은 아니지만 영향을 받는 시스템이 요청 수신을 중단하도록 만들 수 있습니다.

이 보안 업데이트의 심각도는 지원되는 모든 Windows XP 및 Windows Server 2003 에디션에 대해 중요이며 지원되는 모든 Windows Vista 및 Windows Server 2008에 대해서는 보통입니다. 자세한 내용은 이 섹션에서 **영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어**를 참조하십시오.

보안 업데이트는 PGM 구문 분석의 패킷 처리 방식을 수정하여 이러한 취약점을 해결합니다. 취약점에 대한 자세한 내용은 **취약점 정보**에서 각 취약점 항목의 자주 제기되는 질문 사항(FAQ)을 참조하십시오.

**권장 사항.**  이 업데이트를 가능한 빨리 적용하는 것이 좋습니다.

**문제점.** 이 보안 업데이트를 설치할 때 고객이 겪을 수 있는 현재까지 알려진 문제점은 [Microsoft 기술 자료 문서 950762](http://support.microsoft.com/kb/950762)에 나와 있습니다. 이 문서는 이러한 문제점에 대한 권장 해결 방법도 소개합니다. 현재 알려진 문제점 및 권장 해결 방법은 이 소프트웨어의 특정 릴리스에만 해당되나, 이 문서는 다른 문서에 대한 링크도 제공합니다.

PGM 잘못된 길이 취약점(CVE-2008-1440)  
-------------------------------------
  
Microsoft Windows XP 및 Windows Server 2003에 구현된 PGM(Pragmatic General Multicast)에 서비스 거부 취약점이 존재합니다. 이 취약점은 특수하게 조작된 PGM 패킷에 대한 부적절한 유효성 확인으로 인해 발생합니다. 취약점 악용에 성공한 공격자는 컴퓨터에서 응답을 중지하거나 자동으로 다시 시작하도록 만들 수 있습니다.
  
이 취약점을 CVE(Common Vulnerabilities and Exposures) 목록의 표준 항목으로 보려면 [CVE-2008-1440 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1440)을 참조하십시오.
  
#### PGM 잘못된 길이 취약점(CVE-2008-1440)에 대한 완화 요소
  
완화란 취약점의 악용 심각도를 낮출 수 있는 설정, 일반적인 구성 또는 최선책, 기본 상태를 지칭합니다. 다음과 같은 완화 요소가 현재 상황에서 유용할 수 있습니다.
  
-   Windows XP 및 Windows Server 2003을 실행하는 시스템에서 PGM(Pragmatic General Multicast)은 MSMQ(Microsoft Message Queuing) 3.0이 설치되어 있는 경우에만 활성화됩니다. MSMQ 서비스는 기본으로 설치되어 있지 않습니다.
  
#### PGM 잘못된 길이 취약점(CVE-2008-1440)에 대한 대안
  
이 취약점에 대한 대안을 확인하지 못했습니다.
  
#### PGM 잘못된 길이 취약점(CVE-2008-1440)에 대한 FAQ
  
**이 취약점을 악용하면 어느 작업까지 가능합니까?**    
이 취약점은 서비스 거부 취약점입니다. 원격으로 이 취약점을 악용한 익명 공격자는 영향을 받는 시스템에서 응답을 중지하거나 다시 시작하도록 만들 수 있습니다. 서비스 거부 취약점으로 인해 공격자가 코드를 실행하거나 사용자 권한이 상승되는 것은 아니지만 영향을 받는 시스템이 요청 수신을 중단하도록 만들 수 있습니다.
  
**취약점의 원인은 무엇입니까?**    
프로토콜의 구문 분석 코드가 PGM 패킷에서 옵션 길이 필드의 유효성을 제대로 검사하지 못합니다. 영향을 받는 시스템에서 잘못된 옵션 길이 필드가 있는 특수하게 조작된 PGM 패킷을 수신하면 구문 분석에서 무한 루프로 진입하며 결과적으로 시스템이 응답을 하지 못하도록 만들 수 있습니다.
  
**PGM(Pragmatic General Multicast)이란 무엇입니까?**  
PGM은 안정적이고 확장 가능한 멀티캐스트 프로토콜로서 수신기가 손실을 탐지하여 손실된 데이터의 재전송을 요청하거나 응용 프로그램에 복구 불가능한 손실을 통보하도록 지원합니다. PGM은 안정적인 수신을 위한 프로토콜입니다. 즉, 수신기가 모든 데이터를 수신하도록 보장함으로써 통신 안정성에 대해 보낸 사람의 수신 책임을 면해줍니다. PGM은 여러 출처에서 여러 수신기로 중복이 없는 멀티캐스트 데이터를 전달해야 하는 응용 프로그램에 적합합니다. PGM는 배달 확인을 지원하지 않으며, 여러 보낸 사람으로부터의 패킷 주문을 보장하지도 않습니다. PGM에 대한 자세한 내용은 [MSDN 문서](http://msdn.microsoft.com/en-us/library/ms740125(vs.85).aspx)를 참조하십시오.
  
**Message Queuing이란 무엇입니까?**  
MSMQ(Microsoft Message Queuing)는 Windows 운영 체제의 구성 요소로서 의도된 수신자 응용 프로그램이 실행되지 않거나 송신자 또는 수신자 응용 프로그램이 실행되고 있는 컴퓨터에서 네트워크 연결이 끊어진 경우에도 협력하는 응용 프로그램은 메시지를 보내고 받을 수 있습니다. 메시지는 목적지 큐에 도달할 때까지 MSMQ에 의해 저장되어 전달됩니다. 이후 수신자 응용 프로그램 실행 시 큐에서 메시지를 검색할 수 있습니다. MSMQ에서 송신자 및 수신자 응용 프로그램을 동시에 실행할 필요가 없도록 둘을 분리합니다. Message Queuing은 기본 제공되는 보안 강화, 트랜잭션 지원 및 기타 기능을 제공합니다. MSMQ에 대한 자세한 내용은 [MSMQ: 자주 제기되는 질문 사항(FAQ) (영문)](http://www.microsoft.com/windowsserver2003/techinfo/overview/msmqfaq.mspx)을 참조하십시오.
  
**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**    
이 취약점을 성공적으로 악용한 공격자는 사용자 시스템이 응답을 하지 못하거나 기능을 복원하기 위하여 다시 시작하도록 만들 수 있습니다.
  
**공격자는 이러한 취약점을 어떻게 악용합니까?**    
공격자는 특수하게 조작한 PGM 패킷을 영향을 받는 시스템에 보내 취약점 악용을 시도할 수 있습니다. 이렇게 되면 패킷을 사용하여 영향을 받는 시스템에서 응답을 중단하고 다시 시작하도록 만들 수 있습니다.
  
**취약점으로 인해 주로 공격 받을 위험이 있는 시스템은 무엇입니까?**    
MSMQ를 설치하였거나 PGM 패킷에 대해 수신 대기하는 PGM 지원 응용 프로그램을 실행하고 있는 워크스테이션과 서버가 주로 이 취약점으로 인한 위험에 노출됩니다.

업데이트는 PGM 패킷의 옵션 길이 필드의 유효성 검사 방식을 수정하여 취약점을 제거합니다.

이 보안 업데이트를 설치할 때 설치 관리자는 해당 컴퓨터에서 하나 이상의 업데이트할 파일이 Microsoft 핫픽스에 의해 이미 업데이트되었는지 확인합니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.  
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.  
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

이 보안 업데이트를 설치할 때 설치 관리자는 해당 컴퓨터에서 하나 이상의 업데이트할 파일이 Microsoft 핫픽스에 의해 이미 업데이트되었는지 확인합니다.

*핫픽스*와 같이 이 공지에서 사용하는 용어에 대한 자세한 내용은 [Microsoft 기술 자료 문서 824684](http://support.microsoft.com/kb/824684)를 참조하십시오.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 검색, 탐지 도구 및 지침 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.  
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.  
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

*핫픽스*와 같이 이 공지에서 사용하는 용어에 대한 자세한 내용은 [Microsoft 기술 자료 문서 824684](http://support.microsoft.com/kb/824684)를 참조하십시오.

-   **파일 버전 확인**
  
    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    3.  **일반** 탭에 표시된 파일 크기를 공지 KB 문서에 제공된 파일 정보 표와 비교합니다.  
    4.  **세부 정보** 탭 및 비교 정보를 클릭하여 파일 버전과 수정된 날짜를 공지 KB 문서에 제공된 파일 정보 표와 비교할 수도 있습니다.  
    5.  마지막으로, **이전 버전** 탭을 클릭하여 표시된 이전 파일 버전의 파일 정보를 새 파일 또는 업데이트된 파일 버전의 파일 정보와 비교할 수도 있습니다.
