Security Bulletin

Microsoft 보안 공지 MS09-053 - 중요
===================================

IIS(인터넷 정보 서비스) FTP 서비스의 취약점으로 인한 원격 코드 실행 문제점(975254)
----------------------------------------------------------------------------------

게시된 날짜: 2009년 10월 14일 수요일 | 업데이트된 날짜: 2009년 10월 19일 월요일

**버전:** 1.1

### 일반 정보

#### 요약

이 보안 업데이트는 Microsoft IIS(인터넷 정보 서비스) 5.0, Microsoft IIS(인터넷 정보 서비스) 5.1, Microsoft IIS(인터넷 정보 서비스) 6.0 및 Microsoft IIS(인터넷 정보 서비스) 7.0의 FTP 서비스에 대해 공개적으로 보고된 취약점 2건을 해결합니다. IIS 7.0에서는 FTP 서비스 6.0만 영향을 받습니다. 이 취약점으로 인해 IIS 5.0에서 FTP 서비스를 실행하고 있는 시스템에 원격 코드 실행(RCE)이 발생하거나 IIS 5.0, IIS 5.1, IIS 6.0 또는 IIS 7.0에서 FTP 서비스를 실행하고 있는 시스템에 서비스 거부(DoS)가 발생할 수 있습니다.

이 보안 업데이트의 심각도는 IIS 5.0, IIS 5.1, IIS 6.0 및 IIS 7.0(FTP 서비스 6.0만 해당)에 대해 중요입니다. 자세한 내용은 이 섹션에서 **영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어**를 참조하십시오.

이 보안 업데이트는 FTP 서비스에서 목록 작업을 처리하는 방식을 수정하여 취약점을 해결합니다. 취약점에 대한 자세한 내용은 다음 섹션, **취약점 정보**에서 각 취약점 항목의 자주 제기되는 질문 사항(FAQ)를 참조하십시오.

이 보안 업데이트는 [Microsoft 보안 권고 975191](http://technet.microsoft.com/security/advisory/975191)에서 처음 설명한 취약점도 해결합니다.

**권장 사항.**  대다수의 고객은 자동 업데이트를 사용하고 있기 때문에 따로 조치를 취할 필요가 없습니다. 이 보안 업데이트가 자동으로 다운로드되고 설치됩니다. 자동 업데이트를 사용하고 있지 않은 고객은 수동으로 업데이트를 확인하고 이 업데이트를 설치해야 합니다. 자동 업데이트의 특정 구성 옵션에 대한 자세한 내용은 [Microsoft 기술 자료 문서 294871](http://support.microsoft.com/kb/294871)을 참조하십시오.

관리자 및 기업 설치의 경우나 이 보안 업데이트를 수동으로 설치하려는 최종 사용자의 경우에는 고객이 업데이트 관리 소프트웨어를 사용하거나 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 서비스를 통해 업데이트를 확인하여 업데이트를 최대한 빨리 적용하는 것이 좋습니다.

이 공지 뒷부분에 있는 **검색, 탐지 도구 및 지침** 섹션도 참조하십시오.

**문제점.** 이 보안 업데이트를 설치할 때 고객이 겪을 수 있는 현재까지 알려진 문제점은 [Microsoft 기술 자료 문서 975254](http://support.microsoft.com/kb/975254)에 나와 있습니다. 이 문서는 이러한 문제점에 대한 권장 해결 방법도 소개합니다.

#### 영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어

다음 소프트웨어는 테스트를 거쳐 영향을 받는 버전 또는 에디션이 확인되었습니다. 다른 버전은 지원 기간이 끝났거나 영향을 받지 않습니다. 사용 중인 소프트웨어 버전에 대한 지원 기간을 확인하려면 [Microsoft 지원 기간 정책 웹 사이트](http://go.microsoft.com/fwlink/?linkid=21742)를 참조하십시오.

**파일 정보 세부 사항은 어디에 있습니까?**  
파일 정보 세부 사항의 위치에 대해서는 **보안 업데이트 배포** 섹션에서 참조 표를 참조하십시오.

**이 업데이트가 여러 가지 보고된 보안 취약점을 해결하는 이유는 무엇입니까?**  
이 업데이트에 여러 가지 취약점에 대한 지원 기능이 포함된 이유는 이러한 문제를 해결하는 데 필요한 수정 사항이 관련 파일에 위치하기 때문입니다. 거의 동일한 여러 가지 업데이트를 설치하는 대신 이 업데이트만 설치하면 됩니다.

**Windows Server 2008 서버 관리자를 사용하여 FTP를 설치했습니다. 이 시스템이 영향을 받습니까?**
Windows Server 2008 서버 관리자를 사용하여 FTP를 설치한 경우, FTP 서비스 6.0이 설치되며 영향을 받습니다. 이 공지에서 해당 보안 업데이트를 설치하거나 FTP 서비스 7.5로 업그레이드하는 것이 좋습니다.

**Windows** **Vista** **또는 Windows Server 2008의 FTP 서비스 7.5가 이 취약점의 영향을 받습니까?**
아니요. 부정기 다운로드를 통해 사용할 수 있는 32비트 및 x64 기반 버전의 Windows Vista 및 Windows Server 2008용 FTP 서비스 7.5는 이 취약점의 영향을 받지 않습니다. 다운로드 센터의 다음 사이트를 참조하십시오.

-   [IIS 7.0(x86)용 Microsoft FTP 서비스 7.5](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=b7f5b652-8c5c-447a-88b8-8cfc5c13f571)
-   [IIS 7.0(x64)용 Microsoft FTP 서비스 7.5](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=ffb7c167-279e-48d3-8169-dea85784c4d1)

**이 보안 공지에서 설명하는 소프트웨어의 이전 버전을 사용하고 있습니다. 어떻게 해야 합니까?**

IIS FTP 서비스 DoS 취약점(CVE-2009-2521)
----------------------------------------

<span></span>
Microsoft IIS(인터넷 정보 서비스) 5.0, Microsoft IIS(인터넷 정보 서비스) 5.1, Microsoft IIS(인터넷 정보 서비스) 6.0 및 Microsoft IIS(인터넷 정보 서비스) 7.0의 FTP 서비스에 취약점이 존재합니다. 이 취약점으로 인해 서비스 거부(DoS)가 발생할 수 있습니다.

이 취약점을 CVE(Common Vulnerabilities and Exposures) 목록의 표준 항목으로 보려면 [CVE-2009-2521 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2521)을 참조하십시오.

#### IIS FTP 서비스 DoS 취약점(CVE-2009-2521)에 대한 완화 요소

완화란 취약점의 악용 심각도를 낮출 수 있는 설정, 일반적인 구성 또는 최선책, 기본 상태를 지칭합니다. 다음과 같은 완화 요소가 현재 상황에서 유용할 수 있습니다.

-   FTP 서비스는 지원 대상인 모든 Microsoft Windows 2000, Windows XP, Windows Vista, Windows Server 2003, Windows Server 2008 및 Windows 7에 기본적으로 설치되지 않습니다.

#### IIS FTP 서비스 DoS 취약점(CVE-2009-2521)에 대한 대안

대안이란 기본적인 취약점을 수정하지는 않지만 업데이트를 적용하기 전에 알려진 공격 경로를 차단하는 데 유용한 설정 또는 구성 변경을 지칭합니다. Microsoft는 다음과 같은 대안을 테스트했으며, 대안으로 인한 기능 저하 여부를 해당 항목에서 설명합니다.

-   **FTP 서비스 사용 안 함**

    다음 절차 중 하나를 사용하여 FTP 서비스를 사용하지 않도록 설정합니다.

    -   명령 프롬프트에서 다음 명령을 입력합니다.
        **net stop msftpsvc**
    -   인터넷 정보 서비스 패널에서 다음 단계를 수행합니다.
        1. 서버를 확장하고 FTP 사이트를 마우스 오른쪽 단추로 클릭합니다.
        2. **중지**를 클릭합니다.

    **대안의 영향.** FTP 서비스를 사용하여 파일을 전송할 수 없게 됩니다.

    **대안 실행 취소 방법.**

    다음 절차 중 하나를 사용하여 FTP 서비스를 다시 사용하도록 설정합니다.

    -   명령 프롬프트에서 다음 명령을 입력합니다.
        **net start msftpsvc**
    -   인터넷 정보 서비스 패널에서 다음 단계를 수행합니다.
        1. 서버를 확장하고 FTP 사이트를 마우스 오른쪽 단추로 클릭합니다.
        2. **시작**을 클릭합니다.

-   **익명 사용자의 FTP 쓰기 액세스 거부**

    이 대안은 익명 사용자의 서비스 거부 공격을 차단합니다.

    이 대안은 알려진 악용을 차단하며 이 공격 경로의 모든 변형으로 인한 영향을 최소화합니다. 그러나 이 대안은 악용을 완전히 사용하지 못하도록 취약점을 렌덩링하지는 못합니다.

    기본적으로 익명 사용자에게는 FTP 쓰기 액세스 권한이 부여되지 않습니다. FTP 서버에서 익명 사용자에게 쓰기 액세스 권한을 부여한 경우 관리자는 IIS 권한을 수정하여 익명 사용자에게 쓰기 액세스 권한을 허용하지 않을 수 있습니다. 이 대안으로 인해 익명 사용자가 원격 코드를 실행할 수 없게 됩니다.

    익명 사용자에게 FTP 쓰기 액세스 권한을 허용하지 않도록 IIS 권한을 수정하려면 다음 단계를 수행하십시오.

    1.  IIS 관리자를 시작합니다.
    2.  **기본 FTP 사이트**를 마우스 오른쪽 단추로 클릭하고 **속성**을 가리킵니다.
    3.  **홈 디렉토리** 탭을 클릭합니다.
    4.  **쓰기**가 선택 해제되어 있는지 확인합니다.

    **대안의 영향:** 익명 사용자가 FTP 서비스를 사용하여 서버에 콘텐츠를 게시할 수 없게 됩니다.

    **대안 실행 취소 방법.**

    1.  IIS 관리자를 시작합니다.
    2.  **기본 FTP 사이트**를 마우스 오른쪽 단추로 클릭하고 **속성**을 가리킵니다.
    3.  **홈 디렉토리** 탭을 클릭합니다.
    4.  **쓰기**가 선택되어 있는지 확인합니다.

-   **익명 사용자의 FTP 액세스 거부**

    이 대안은 유효한 사용자 계정이 없는 인증되지 않은 사용자의 서비스 거부 공격을 차단합니다.

    이 대안은 알려진 악용을 차단하며 이 공격 경로의 모든 변형으로 인한 영향을 최소화합니다. 그러나 이 대안은 악용을 완전히 사용하지 못하도록 취약점을 렌덩링하지는 못합니다.

    기본적으로 익명 사용자에게는 FTP 쓰기 액세스 권한이 부여되지 않습니다. FTP 서버에서 익명 사용자에게 쓰기 액세스 권한을 부여한 경우 관리자는 IIS 권한을 수정하여 익명 사용자에게 쓰기 액세스 권한을 허용하지 않을 수 있습니다. 이 대안으로 인해 익명 사용자가 취약점을 악용할 수 없게 됩니다.

    익명 사용자에게 FTP 쓰기 액세스 권한을 허용하지 않도록 IIS 권한을 수정하려면 다음 단계를 수행하십시오.

    1.  IIS 관리자를 시작합니다.
    2.  **기본 FTP 사이트**를 마우스 오른쪽 단추로 클릭하고 **속성**을 가리킵니다.
    3.  **보안 계정** 탭을 클릭합니다.
    4.  **익명 연결 허용**이 선택 해제되어 있는지 확인합니다.

    **대안의 영향:** 익명 사용자가 FTP 서비스에 액세스할 수 없게 됩니다.

    **대안 실행 취소 방법.**

    1.  IIS 관리자를 시작합니다.
    2.  **기본 FTP 사이트**를 마우스 오른쪽 단추로 클릭하고 **속성**을 가리킵니다.
    3.  **보안 계정** 탭을 클릭합니다.
    4.  **익명 연결 허용**이 선택되어 있는지 확인합니다.

-   **FTP 사용자가 디렉터리를 생성하지 못하도록 NTFS 파일 시스템 권한 수정**

    이 대안은 인증된 사용자의 서비스 거부 공격을 차단합니다.

    이 대안은 알려진 악용을 차단하며 이 공격 경로의 모든 변형으로 인한 영향을 최소화합니다. 그러나 이 대안은 악용을 완전히 사용하지 못하도록 취약점을 렌덩링하지는 못합니다.

    관리자는 서버에서 호스팅하는 FTP 사이트의 루트 디렉터리에서 FTP 사용자가 디렉터리를 생성하지 못하도록 NTFS 파일 시스템 권한을 수정할 수 있습니다. 이 수정 사항을 적용해도 FTP 사용자는 여전히 기존 디렉터리에 파일을 업로드할 수 있습니다. 이 대안으로 인해 인증된 사용자 및 인증되지 않은 사용자가 원격 코드를 실행할 수 없게 됩니다.

    관리자는 다음 단계를 수행하여 사용자 그룹의 디렉터리 생성 권한을 제거할 수 있습니다. FTP 사용자를 관리하기 위해 구성된 FTP 사용자 또는 사용자 지정 그룹이 있는 경우 다음 5단계의 사용자 그룹을 사용자 지정 ID로 변경합니다.

    2.  디렉터리를 마우스 오른쪽 단추로 클릭한 후 **속성**을 선택합니다.
    3.  **보안** 탭을 클릭하고 **고급**을 클릭합니다.
    4.  **사용 권한 변경**을 클릭합니다.
    5.  **사용자** 그룹을 선택하고 **편집**을 클릭합니다.
    6.  **폴더 만들기/데이터 추가**의 선택을 해제합니다.

    **대안의 영향:** FTP 사용자가 FTP 서비스를 통해 디렉터리를 만들지 못합니다. FTP 사용자는 FTP 서비스를 통해 여전히 기존 디렉터리에 파일을 업로드할 수 있습니다.

#### IIS FTP 서비스 DoS 취약점(CVE-2009-2521)에 대한 FAQ

**이 취약점을 악용하면 어느 작업까지 가능합니까?**  
이 취약점은 서비스 거부 취약점입니다. 공격자는 이 취약점을 악용하여 영향을 받는 서비스에서 응답을 중지하거나 다시 시작하도록 만들 수 있습니다. 서비스 거부 취약점으로 인해 공격자가 코드를 실행하거나 사용자 권한이 상승되는 것은 아니지만 영향을 받는 시스템이 요청 수신을 중단하도록 만들 수 있습니다.

**취약점의 원인은 무엇입니까?**  
IIS의 Microsoft FTP 서비스가 목록 명령을 처리하는 방식으로 인해 취약점이 발생합니다.

**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**  
이 취약점을 악용한 공격자는 사용자 컴퓨터가 응답을 하지 못하거나 다시 시작하도록 만들 수 있습니다.

**공격자는 이러한 취약점을 어떻게 악용합니까?**  
공격자는 특수하게 조작한 목록 명령을 서버로 보내 이 취약점을 악용할 수 있습니다.

**취약점으로 인해 주로 공격 받을 위험이 있는 시스템은 무엇입니까?**  
Microsoft FTP(파일 전송 프로토콜) 서비스를 실행하는 서버 시스템입니다.

**업데이트는 어떤 기능을 수행합니까?**  
이 업데이트는 Microsoft FTP 서버가 목록 명령 작업을 처리하는 방식을 수정합니다.

**이 보안 공지를 게시했을 때 이 취약점이 공개되어 있었습니까?**  
예. 이 취약점은 공개되었습니다. 이 취약점에는 CVE(Common Vulnerability and Exposure) 번호 [CVE-2009-2521 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2521)이 지정되었습니다.

    1.  FTP 사이트의 루트 디렉터리를 찾습니다. 기본적으로 루트 디렉터리의 위치는 %systemroot%\\inetpub\\ftproot입니다.
    2.  디렉터리를 마우스 오른쪽 단추로 클릭한 후 **속성**을 선택합니다.
    3.  **보안** 탭을 클릭하고 **고급**을 클릭합니다.
    4.  **사용 권한 변경**을 클릭합니다.
    5.  **사용자** 그룹을 선택하고 **편집**을 클릭합니다.
    6.  **폴더 만들기/데이터 추가**를 선택합니다.

-   **FTP 서비스 7.5로 업그레이드**

    Internet Information Services 7.0에 대해 FTP 서비스 7.5를 사용할 수 있습니다. 이 버전의 FTP 서비스는 이 보안 공지에서 설명한 취약점의 영향을 받지 않습니다. 다운로드 센터의 다음 사이트를 참조하십시오.

    -   [IIS 7.0(x86)용 Microsoft FTP 서비스 7.5](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=b7f5b652-8c5c-447a-88b8-8cfc5c13f571)
    -   [IIS 7.0(x64)용 Microsoft FTP 서비스 7.5](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=ffb7c167-279e-48d3-8169-dea85784c4d1)

    **대안의 영향:** FTP 사이트를 FTP 서비스 6.0에서 FTP 서비스 7.5로 마이그레이션해야 합니다.

#### IIS FTP 서비스 RCE 및 DoS 취약점(CVE-2009-3023)에 대한 FAQ

**이 취약점을 악용하면 어느 작업까지 가능합니까?**  
이 취약점은 서비스 거부 취약점입니다. 공격자는 이 취약점을 악용하여 영향을 받는 서비스에서 응답을 중지하거나 다시 시작하도록 만들 수 있습니다. 서비스 거부 취약점으로 인해 공격자가 코드를 실행하거나 사용자 권한이 상승되는 것은 아니지만 영향을 받는 시스템이 요청 수신을 중단하도록 만들 수 있습니다.

이 취약점은 Microsoft Windows 2000 서비스 팩 4를 실행하는 시스템의 경우 원격 코드 실행 취약점이고, 기타 모든 영향을 받는 플랫폼의 경우 서비스 거부 취약점입니다. Microsoft Windows 2000 서비스 팩 4의 IIS 5.0에서 이 취약점 악용에 성공한 공격자는 LocalSystem 컨텍스트에서 원격 코드 실행을 유발할 수 있습니다. IIS 5.1 및 IIS 6.0이 취약점 악용에 성공한 공격자는 IIS FTP 서비스를 중단시키거나 서비스가 응답하지 않도록 할 수 있습니다.

**취약점의 원인은 무엇입니까?**  
IIS의 Microsoft FTP 서비스가 목록 명령을 처리하는 방식으로 인해 취약점이 발생합니다.

**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**  
FTP 서비스에 액세스 권한이 있는 공격자는 이 취약점을 사용하여 IIS 5.0을 실행하고 있는 시스템에서 스택 기반 오버런을 발생시켜 LocalSystem 계정의 컨텍스트에서 임의 코드를 실행하거나 IIS 5.1 또는 IIS 6.0을 실행하는 영향을 받는 시스템에서 서비스 거부를 유발할 수 있습니다. LocalSystem 계정에 대한 자세한 내용은 MSDN 문서, [LocalSystem 계정(영문)](http://msdn.microsoft.com/en-us/library/ms684190(vs.85).aspx)을 참조하십시오.

**공격자는 이러한 취약점을 어떻게 악용합니까?**  
FTP 서비스에 대한 액세스 권한이 있는 익명 사용자 또는 인증된 사용자는 FTP NLST 명령을 보내 메모리를 손상시키고, Microsoft Windows 2000에서 IIS 5.0을 실행하는 시스템에서 원격 코드를 실행하거나 다른 플랫폼에서 IIS 5.1 또는 IIS 6.0을 실행하는 시스템에서 서비스 거부를 유발할 수 있습니다.

**취약점으로 인해 주로 공격 받을 위험이 있는 시스템은 무엇입니까?**  
Microsoft FTP(파일 전송 프로토콜) 서비스를 실행하는 서버 시스템입니다.

**업데이트는 어떤 기능을 수행합니까?**  
이 업데이트는 Microsoft FTP 서버가 목록 명령 작업을 처리하는 방식을 수정합니다.

**이 보안 공지를 게시했을 때 이 취약점이 공개되어 있었습니까?**  
예. 이 취약점은 공개되었습니다. 이 취약점에는 CVE(Common Vulnerability and Exposure) 번호 [CVE-2009-3023](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3023)이 지정되었습니다.

**이 보안 공지를 게시했을 때 Microsoft는 이 취약점이 악용되었다는 보고를 받았습니까?**  
예. Microsoft는 이 취약점을 악용하려는 제한적이며 대상이 일정한 공격에 대한 보고를 받았습니다.

**이 보안 업데이트를 적용하면 취약점을 악용하려는 일반에 공개된 코드로부터 고객을 보호할 수 있습니까?**  
예. 이 보안 업데이트는 현재 악용되고 있는 취약점을 해결합니다. 해결된 취약점에 CVE(Common Vulnerability and Exposure) 번호 [CVE-2009-3023](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3023)이 지정되었습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 검색, 탐지 도구 및 지침 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.
