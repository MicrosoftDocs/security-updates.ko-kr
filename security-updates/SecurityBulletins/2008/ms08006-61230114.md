Security Bulletin

Microsoft 보안 공지 MS08-006 - 중요
===================================

IIS(인터넷 정보 서비스)의 취약점으로 인한 원격 코드 실행 문제점(942830)
-----------------------------------------------------------------------

게시된 날짜: 2008년 2월 13일 수요일 | 업데이트된 날짜: 2008년 2월 21일 목요일

**버전:** 1.1

### 일반 정보

#### 요약

이 중요 업데이트는 비공개적으로 보고된 IIS(인터넷 정보 서비스)의 취약점 1건을 해결합니다. IIS에서 ASP 웹 페이지에 대한 입력을 처리하는 방식에 원격 코드 실행 취약점이 존재합니다. 이 취약점 악용에 성공한 공격자는 IIS 서버에서 WPI(Worker Process Identity)와 동일한 권한으로 작업을 수행할 수 있습니다. WPI는 기본적으로 네트워크 서비스 계정 권한으로 구성됩니다. 사용자 권한을 가진 계정을 사용하는 WPI로 응용 프로그램 풀이 구성되어 있는 ASP 페이지가 포함된 IIS 서버는 기본 WPI 설정으로 응용 프로그램 풀이 구성되어 있는 IIS 서버보다 더 심각한 영향을 받습니다.

보안 업데이트의 심각도는 지원되는 모든 Windows XP 및 Windows Server 2003의 Microsoft IIS(인터넷 정보 서비스)에 대해 중요입니다. 자세한 내용은 **영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어**를 참조하십시오.

이러한 취약점에 대한 자세한 내용은 다음 섹션, **취약점 정보**에서 취약점 항목의 자주 제기되는 질문 사항(FAQ)을 참조하십시오.

**권장 사항:** 이 업데이트를 가능한 빨리 적용하는 것이 좋습니다.

**알려진 문제점.** 없음

ASP 취약점 - CVE-2008-0075  
--------------------------
  

IIS(인터넷 정보 서비스)에서 ASP 웹 페이지에 대한 입력을 처리하는 방식에 원격 코드 실행 취약점이 존재합니다. 공격자는 웹 사이트의 ASP 페이지에 악의적인 입력을 전달하여 취약점을 악용할 수 있습니다. 이 취약점 악용에 성공한 공격자는 네트워크 서비스 계정 권한이 기본적으로 구성되어 있는 WPI(Worker Process Identity)와 동일한 권한으로 IIS 서버에서 어떠한 작업도 수행할 수 있습니다.
  
이 취약점을 CVE(Common Vulnerabilities and Exposures) 목록의 표준 항목으로 보려면 [CVE-2008-0075 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0075)를 참조하십시오.
  
#### ASP 취약점 - CVE-2008-0075에 대한 완화 요소
  
완화란 취약점의 악용 심각도를 낮출 수 있는 설정, 일반적인 구성 또는 최선책, 기본 상태를 지칭합니다. 다음과 같은 완화 요소가 현재 상황에서 유용할 수 있습니다.
  
-   IIS 5.1은 Windows XP Professional 서비스 팩 2의 기본 설치에 속하지 않습니다.  
-   지원되는 Windows Server 2003 에디션에서 IIS는 기본적으로 설치되거나 사용하도록 설정되지 않습니다.  
-   지원되는 Windows Server 2003 에디션에서 IIS를 사용하도록 설정한 경우 클래식 ASP는 기본적으로 설치되거나 사용하도록 설정되지 않습니다.  
-   지원되는 Windows Server 2003 에디션에서 IIS를 사용하도록 설정하고 클래식 ASP가 사용되는 경우 공격자는 이 취약점을 악용해도 기본적으로 Network Service 계정 권한만 획득할 수 있습니다. 기본적으로 Network Service 계정 권한은 인증된 사용자의 사용자 권한과 동일합니다.  
-   ASP.NET은 이 취약점의 영향을 받지 않습니다. ASP.NET만 설치되어 있는 경우 ASP는 이 취약점으로 인한 위험에 노출되지 않습니다.
  
#### ASP 취약점 - CVE-2008-0075에 대한 대안
  
대안이란 기본적인 취약점을 수정하지는 않지만 업데이트를 적용하기 전에 알려진 공격 경로를 차단하는 데 유용한 설정 또는 구성 변경을 지칭합니다. Microsoft는 다음과 같은 대안을 테스트했으며, 대안으로 인한 기능 저하 여부를 해당 항목에서 설명합니다.
  
-   **Windows Server 2003의 경우 클래식 ASP 사용 안 함**
  
    클래식 ASP를 사용하지 않도록 설정하려면 다음 단계를 따르십시오.
  
    1. **시작**, **관리 도구**, **IIS(인터넷 정보 서비스) 관리자**를 차례로 클릭합니다.
  
    2. 서버 이름 옆의 **더하기 기호**(+)를 클릭하고 **웹 서비스 확장** 폴더를 클릭합니다.
  
    3. 오른쪽 창의 **Active Server Pages**를 클릭하고 **금지**를 클릭합니다.
  
    **대안의 영향.** 클래식 ASP 페이지를 사용할 수 없게 됩니다.
  
    **대안 실행 취소 방법.** 대안을 실행 취소하려면 위에 있는 1~3단계를 수행하고 **Active Server Pages**를 선택한 다음 **허용**을 클릭합니다.
  
#### ASP 취약점 - CVE-2008-0075에 대한 FAQ
  
**이 취약점을 악용하면 어느 작업까지 가능합니까?**  
이것은 원격 코드 실행 취약점입니다. 이 취약점 악용에 성공한 공격자는 네트워크 서비스 계정 권한이 기본적으로 구성되어 있는 WPI(Worker Process Identity)와 동일한 권한으로 IIS 서버에서 작업을 수행할 수 있습니다.
  
**취약점의 원인은 무엇입니까?**  
취약점의 원인은 ASP 페이지에 대한 입력을 부정확하게 처리하는 IIS(인터넷 정보 서비스)입니다.
  
**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**  
이 취약점 악용에 성공한 공격자는 네트워크 서비스 계정 권한이 기본적으로 구성되어 있는 WPI(Worker Process Identity)와 동일한 권한으로 IIS 서버에서 작업을 수행할 수 있습니다. Network Service 계정 권한으로 구정된 서비스는 인증된 사용자 수준 액세스 권한만 가져올 수 있으며, 관리자 수준 액세스 권한은 가져올 수 없습니다. 자세한 내용은 [NetworkService 계정 (영문)](http://msdn2.microsoft.com/ko-kr/library/ms684272(vs.85).aspx)을 참조하십시오. 사용자 권한을 가진 계정을 사용하는 WPI로 응용 프로그램 풀이 구성되어 있는 IIS 서버는 기본 WPI 설정으로 응용 프로그램 풀이 구성되어 있는 IIS 서버보다 더 심각한 영향을 받습니다.
  
네트워크 서비스 계정은 원격 인증을 수행할 때 컴퓨터의 자격 증명을 사용하지만, 서버 자체에서의 권한 수준은 상당히 낮으며 로컬 관리자 권한이 없습니다. 자세한 내용은 [NetworkService Account(NetworkService 계정) (영문)](http://msdn2.microsoft.com/ko-kr/library/ms684272(vs.85).aspx), [IIS and Built-in Accounts (IIS 6.0)(IIS 및 내장 계정((IIS 6.0)) (영문)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/3648346f-e4f5-474b-86c7-5a86e85fa1ff.mspx), [Configuring Worker Process Identities (IIS 6.0)(WPI 구성(IIS 6.0)) (영문)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/76be66e2-1db1-432e-95be-14ff59c2ec75.mspx) 및 [Configuring Application Pools in IIS 6.0 (IIS 6.0)(IIS 6.0 응용 프로그램 풀 구성(IIS 6.0)) (영문)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/2a231dcb-d786-4b6d-b3ba-bda04061a5e7.mspx)을 참조하십시오.
  
**공격자는 이러한 취약점을 어떻게 악용합니까?**  
공격자는 ASP 페이지에 특수하게 조작된 입력을 전달할 수 있습니다. 해당 ASP 페이지는 이미 존재하고 있거나 호스트된 환경에서 사용자가 업로드했을 수 있습니다.
  
**IIS용 작업자 프로세스란 무엇입니까?**  
작업자 프로세스는 웹 서버에 대한 서버측 처리 작업을 실행하는 프로세스입니다**.** 작업자 프로세스 및 IIS 6.0에서 구현 방법에 자세한 내용은 [Worker Processes (IIS 6.0)(작업자 프로세스(IIS 6.0)) (영문)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/29f53968-0115-451f-b26d-5ad58d87b5d1.mspx)를 참조하십시오. 작업자 프로세스 및 격리 상태에서의 웹 기반 응용 프로그램 실행 방법에 대한 자세한 내용은 [Worker Process Isolation Mode (IIS 6.0)(작업자 프로세스 격리 모드(IIS 6.0)) (영문)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/333f3410-b566-49bc-8399-57bcc404db8c.mspx)를 참조하십시오.
  
**ASP란 무엇입니까?**  
Microsoft ASP(Active Server Pages)는 동적 및 대화형 웹 응용 프로그램을 만드는 데 사용될 수 있는 서버측 스크립팅 기술입니다. ASP 페이지는 사용자의 브라우저에 전송되기 전에 웹 서버에서 처리하는 서버측 스크립트가 포함된 HTML 페이지입니다.
  
**취약점으로 인해 주로 공격받을 위험이 있는 시스템은 무엇입니까?**  
IIS 및 ASP를 사용하도록 설정한 경우 Windows Server 2003 및 Windows XP Professional 웹 서버가 위험합니다. 고객이 자신의 호스트된 웹 사이트에 ASP 페이지를 업로드하도록 허용하는 웹 호스팅 공급업체도 높은 위험에 노출됩니다.
  
**업데이트는 어떤 기능을 수행합니까?**  
업데이트는 IIS(인터넷 정보 서비스)에서 ASP 페이지에 대한 입력을 처리하는 방식을 변경하여 취약점을 제거합니다.

아니요. Microsoft는 신뢰할 수 있는 정보 공개를 통해 이 취약점 관련 정보를 입수하였습니다. Microsoft는 이 보안 공지를 처음에 게시했을 때 이 취약점이 공개되었다는 정보를 확인하지 못했습니다.
  
**이 보안 공지를 게시했을 때 Microsoft는 이 취약점이 악용되었다는 보고를 받았습니까?**

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 검색, 탐지 도구 및 지침 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.  

        **참고** 설치된 프로그램 또는 운영 체제 버전에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 컴퓨터에 설치된 파일 버전을 확인합니다.   

        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.   

        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.
