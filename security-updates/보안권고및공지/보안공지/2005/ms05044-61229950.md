Security Bulletin

Microsoft 보안 공지 MS05-044 - 보통
===================================

Windows FTP 클라이언트의 취약점으로 인한 파일 전송 위치 변조 문제점 (905495)
----------------------------------------------------------------------------

게시된 날짜: 2005년 10월 11일 화요일 | 업데이트된 날짜: 2005년 10월 26일 수요일

**버전:** 1.0

**게시된 날짜:** 2005년 10월 12일
**버전:** 1.0

#### 요약

**이 내용을 읽어야 할 사람:** Microsoft Windows를 사용하는 고객

**취약점으로 인한 영향:** 변조

**최대 심각도:** 보통

**권장 사항:** 고객은 보안 업데이트 적용을 고려해야 합니다.

**보안 업데이트 대체:** 없음

**주의 사항:** 없음

**테스트된 소프트웨어 및 보안 업데이트 다운로드 위치:**

**영향을 받는 소프트웨어:**

-   Microsoft Windows XP 서비스 팩 1 - [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=351c63a3-ab62-418d-8678-3af791d73a29&displaylang=ko)
-   Microsoft Windows Server 2003 - [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=4940cf64-e1fd-4e88-8980-3106be03bf12&displaylang=ko)
-   Itanium 기반 시스템용 Microsoft Windows Server 2003 - [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=b715147b-de2d-4f14-9548-aff18641d0f3)

**영향을 받는 구성 요소:**

-   Microsoft Windows 2000 서비스 팩 4의 Internet Explorer 6 서비스 팩 1 - [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=fcea60e5-9ea8-4216-ba4d-c85054892dbb&displaylang=ko)

**영향을 받지 않는 소프트웨어:**

-   Microsoft Windows XP 서비스 팩 2
-   Microsoft Windows XP Professional x64 Edition

-   Microsoft Windows Server 2003 SP1 (Itanium 기반 시스템용)
-   Microsoft Windows Server 2003 x64 Edition
-   Microsoft Windows 98, Microsoft Windows 98 Second Edition (SE) 및 Microsoft Windows Millennium Edition (ME)

**영향을 받지 않는 구성 요소:**

-   Microsoft Windows 2000 서비스 팩 4의 Internet Explorer 5.01 서비스 팩 4

위에 나열된 소프트웨어는 테스트를 거쳐 해당 버전이 영향을 받는지 확인되었습니다. 그 외 다른 버전은 더 이상 보안 업데이트가 지원되지 않거나 영향을 받지 않을 수도 있습니다. 사용 중인 제품과 버전에 대한 지원 주기를 확인하려면 [Microsoft 지원 주기 정책 웹 사이트](http://go.microsoft.com/fwlink/?linkid=21742)를 참조하십시오.

**Enterprise Update Scanning Tool(EST)이란 무엇입니까?**
이 사이트에서 제공하는 보안 업데이트를 확인하는 도구를 제공하기 위한 지속적인 노력의 일환으로, Microsoft는 MBSA(Microsoft Baseline Security Analyzer) 및 ODT(Office Detection Tool)에서 MSRC 릴리스 주기에 따라 요구되는 업데이트를 탐지하지 못하는 경우 독립 실행형 탐지 도구를 제공하고 있습니다. 이 독립 실행형 도구를 Enterprise Update Scanning Tool(EST)이라고 하며 기업 관리자용으로 설계되었습니다. 특정 공지에 대해 Enterprise Update Scanning Tool의 버전이 만들어지면 고객은 명령줄 인터페이스(CLI)에서 도구를 실행하고 XML 출력 파일로 결과를 볼 수 있습니다. 고객이 도구를 잘 활용할 수 있도록 자세한 설명서가 도구와 함께 제공됩니다. 또한 SMS 관리자를 위한 통합된 기능을 제공하는 도구 버전도 있습니다.

**Enterprise Update Scanning Tool(EST)을 사용하여 이 업데이트가 필요한지 확인할 수 있습니까?**
예. Microsoft는 이 업데이트를 적용해야 하는지 확인할 수 있는 EST 버전을 개발했습니다. 이번 달에 릴리스되는 EST 버전에 대한 자세한 내용과 다운로드 링크는 다음 Microsoft 웹 사이트를 방문하십시오. SMS 고객은 이 공지 FAQ 항목의 "SMS(Systems Management Server)를 사용하여 이 업데이트가 필요한지 확인할 수 있습니까?" 질문에서 자세한 SMS 및 EST 관련 정보를 검토해야 합니다.

**MBSA(Microsoft Baseline Security Analyzer) 2.0을 사용하여 이 업데이트가 필요한지 확인할 수 있습니까?**
예. MBSA 2.0을 사용하면 이 업데이트가 필요한지 여부를 확인할 수 있습니다. MBSA 2.0은 Microsoft Update가 지원하는 제품용 보안 업데이트를 검색할 수 있습니다. MBSA에 대한 자세한 내용은 [MBSA 웹 사이트](http://www.microsoft.com/korea/technet/security/tools/mbsahome.asp)를 참조하십시오.

**SMS(Systems Management Server)를 사용하여 이 업데이트가 필요한지 확인할 수 있습니까?**
예. SMS를 사용하면 이 보안 업데이트를 검색하고 배포하는 데 도움이 됩니다. SMS는 검색을 위해 MBSA를 사용하므로 앞에서 설명한 MBSA가 검색하지 않는 프로그램과 관련된 동일한 제한이 있습니다.
그러나 SMS Extended Security Update Inventory Tool은 SMS 관리자를 위한 기능이 통합된 EST 버전입니다. 이 EST 버전을 다운로드하려면 다음 Microsoft 웹 사이트를 방문하십시오. 이 도구는 SMS 웹 사이트에서 구할 수도 있습니다. SMS에서 Security Update Inventory Tool을 사용하여 Windows Update에서 제공하고 Software Update Services에서 지원하는 보안 업데이트와 MBSA 1.2.1에서 지원하는 다른 보안 업데이트를 검색할 수 있습니다. Security Update Inventory Tool에 대한 자세한 내용은 다음 Microsoft 웹 사이트를 방문하십시오. Security Update Inventory Tool의 제한 사항에 대한 자세한 내용은 Microsoft 기술 자료 문서 306460을 참조하십시오. SMS에서 Microsoft Updates용 SMS 2003 Inventory Tool을 사용하여 Microsoft Update에서 제공하고, Windows Server Update Services에서 지원하는 보안 업데이트를 검색할 수 있습니다. Microsoft Updates용 SMS 2003 Inventory Tool에 대한 자세한 내용은 다음 Microsoft 웹 사이트를 참조하십시오. SMS에 대한 자세한 내용은 SMS 웹 사이트를 참조하십시오.

취약점 세부 정보
----------------

<span></span>
#### FTP 클라이언트 취약점 - CAN-2005-2126:

Windows FTP 클라이언트에 변조 취약점이 존재합니다. 이 취약점을 통해 공격자는 고객이 FTP를 사용한 파일 전송을 수동으로 선택할 때 파일 전송의 의도된 대상 위치를 수정할 수 있습니다. 이 취약점을 통해 공격자가 영향을 받는 시스템에서 임의의 파일 시스템에 파일을 쓸 수 있습니다.

#### FTP 클라이언트 취약점 - CAN-2005-2126에 대한 완화 요소:

-   공격자가 특수하게 조작된 이름의 파일을 호스팅하는 FTP 서버에 사용자가 방문하도록 성공적으로 유인한 경우에도 파일을 임의로 전송할 수는 없습니다. 영향을 받는 시스템으로 파일을 전송하려면 사용자가 개입해야 합니다.
-   악의적인 FTP 전송이 영향을 받는 시스템에 있는 기존 파일을 덮어쓰려고 시도하는 경우 “파일 덮어쓰기” 경고 메시지가 나타납니다. 경고 메시지가 나타날 때 사용자가 저장을 허용하도록 선택해야만 해당 파일이 저장됩니다.
-   기본적으로 영향을 받는 모든 운영 체제 버전에서 Internet Explorer의 “FTP 사이트에 대한 폴더 보기 사용” 설정은 사용하지 않음으로 되어 있습니다. 이러한 공격은 영향을 받는 시스템에서 사용자가 Internet Explorer의 “FTP 사이트에 대한 폴더 보기 사용” 설정을 직접 사용하도록 설정한 경우에만 성공할 수 있습니다.

#### FTP 클라이언트 취약점 - CAN-2005-2126에 대한 대안:

Microsoft는 다음 대안을 테스트했습니다. 이러한 대안은 취약점 자체를 해결하는 것은 아니지만 알려진 공격을 차단하는 데 도움이 됩니다. 대안으로 인해 기능이 저하되는 경우는 아래 항목에 별도로 표시하였습니다.

-   **신뢰할 수 없는 FTP 서버에서 파일을 다운로드하지 마십시오.**

    신뢰할 수 있는 FTP 서버에서만 파일을 다운로드하면 이 공격의 위험을 줄일 수 있습니다.

#### FTP 클라이언트 취약점 - CAN-2005-2126에 대한 FAQ:

**이 취약점을 악용하면 어느 작업까지 가능합니까?**
이 취약점은 변조 취약점입니다. 이 취약점을 통해 공격자는 고객이 FTP를 사용한 파일 전송을 수동으로 선택할 때 파일 전송의 의도된 대상 위치를 수정할 수 있습니다.

**취약점의 원인은 무엇입니까?**
Windows FTP 클라이언트는 FTP 서버에서 받은 파일 이름을 제대로 확인하지 않습니다.

**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**
이 취약점을 악용한 공격자는 영향을 받는 시스템의 특정 위치에 파일을 저장할 수 있습니다. 이러한 파일은 또 다른 공격을 유도할 수 있습니다. 예를 들어 공격자는 “시작” 폴더에 실행 파일을 저장할 수 있습니다. 그리고 전송된 파일은 나중에 사용자가 로그온하면 실행될 수 있습니다.

**누가 취약점을 악용할 수 있습니까?**
사용자가 특수하게 조작된 파일 이름을 가진 파일을 호스팅하는 FTP 서버를 방문하여 파일을 전송하도록 유인할 수 있는 모든 사람이 취약점을 악용할 수 있습니다.

**공격자는 이러한 취약점을 어떻게 악용합니까?**
FTP 서버에서 특수하게 조작한 파일 이름을 가진 파일을 호스팅함으로써 공격자는 이 취약점을 악용할 수 있습니다. 파일 이름은 FTP 클라이언트에서 제공하는 파일 이름 유효성 검사를 위회하는 방식으로 파일 이름이 구성되고, 사용자 컴퓨터의 유효한 위치로 매핑됩니다. 그런 다음 공격자는 사용자가 이 파일을 다운로드하도록 유인할 수 있습니다.

**FTP 서버를 방문하면 이 취약점이 자동으로 악용됩니까?**
아니요. FTP를 사용하여 파일이 전송 및 저장되려면 사용자 개입이 필요합니다.

**취약점으로 인해 주로 공격 받을 위험이 있는 시스템은 무엇입니까?**
취약점을 악용하려면 사용자가 FTP 서버에 연결하고 FTP 서버에서 파일을 전송해야 합니다. 그러므로 워크스테이션과 같이 FTP 전송이 자주 발생하는 시스템이 이 취약점으로 인해 가장 위험할 수 있습니다. 일반적으로 FTP 서버를 방문하는 데 사용되지 않는 시스템(예: 대부분의 서버 시스템)은 공격 위험이 적습니다.

**업데이트는 어떤 기능을 합니까?**
업데이트는 Windows FTP 클라이언트가 FTP 서버에서 받은 파일 이름을 확인하는 방법을 수정함으로써 취약점을 제거합니다.

**이 보안 공지를 게시했을 때 이 취약점이 공개되었습니까?**
예. 이 취약점은 공개되었습니다. 이 취약점에는 CVE(Common Vulnerability and Exposure) 번호 CAN-2005-2126이 지정되었습니다.

**이 보안 공지를 게시했을 때 Microsoft는 이 취약점이 악용되었다는 보고를 받았습니까?**
아니요. Microsoft는 이 취약점이 공개적으로 고객을 공격하는 데 사용되었다는 정보를 입수하지 못했습니다. 그러나 이 보안 공지가 처음 게시되었을 때 개념 코드의 예제 증거가 공개되었습니다.

**이 보안 업데이트를 적용하면 이 취약점을 악용하는 공개된 코드로부터 고객이 보호를 받을 수 있습니까?**
예. 이 보안 업데이트는 일반에 공개된 개념 증명 코드(proof of concept code)에 의해 밝혀진 취약점을 다룹니다.
