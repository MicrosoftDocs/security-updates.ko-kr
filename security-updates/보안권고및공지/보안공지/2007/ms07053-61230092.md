Security Bulletin

Microsoft 보안 공지 MS07-053 - 중요
===================================

UNIX용 Windows 서비스 취약점으로 인한 권한 상승 문제점(939778)
--------------------------------------------------------------

게시된 날짜: 2007년 9월 11일 화요일 | 업데이트된 날짜: 2007년 9월 19일 수요일

**버전:** 1.1

### 일반 정보

#### 개요

이 중요 보안 업데이트는 일반에 공개된 취약점 1건을 해결합니다. UNIX용 Windows 서비스 3.0, UNIX용 Windows 서비스 3.5 및 UNIX 기반 응용 프로그램용 하위 시스템에 취약점이 존재하며, 특정 setuid 바이너리 파일을 실행하는 경우 공격자 권한 상승이 발생할 수 있습니다.

지원되는 Windows 2000, Windows Server 2003, UNIX용 Windows 서비스 3.0, UNIX용 Windows 서비스 3.5 그리고 Windows Server 2003 및 Windows Vista의 구성 요소인 UNIX 기반 응용 프로그램용 하위 시스템에 대한 중요 보안 업데이트입니다. 자세한 내용은 이 섹션에서 **영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어**를 참조하십시오.

취약점에 대한 자세한 내용은 다음 섹션, **취약점 정보**에서 각 취약점 항목의 자주 제기되는 질문 사항(FAQ)를 참조하십시오.

**권장 사항:** 고객은 보안 업데이트를 적용하는 것이 좋습니다.

**알려진 문제.** 없습니다.

#### 영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어

여기에 나열된 소프트웨어는 테스트를 거쳐 해당 버전이 영향을 받는지 확인되었습니다. 다른 버전은 지원 기간이 끝났거나 영향을 받지 않습니다. 사용 중인 소프트웨어 버전에 대한 지원 기간을 확인하려면 [Microsoft 지원 기간 정책 웹 사이트](http://go.microsoft.com/fwlink/?linkid=21742)를 참조하십시오.

UNIX용 Windows 서비스 권한 상승 문제점 - CVE-2007-3036  
------------------------------------------------------
  
<span></span>
UNIX용 Windows 서비스 3.0, UNIX용 Windows 서비스 3.5 및 UNIX 기반 응용 프로그램용 하위 시스템에 취약점이 존재하며, 특정 setuid 바이너리 파일을 실행하는 경우 공격자 권한 상승이 발생할 수 있습니다. 이 취약점 악용에 성공한 공격자는 권한을 상승시킬 수 있습니다.
  
이 취약점을 CVE(Common Vulnerabilities and Exposures) 목록의 표준 항목으로 보려면 [CVE-2007-3036 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3036)을 참조하십시오.
  
#### UNIX용 Windows 서비스 권한 상승 문제점 - CVE-2007-3036에 대한 완화 요소
  
완화란 취약점의 악용 심각도를 낮출 수 있는 설정, 일반적인 구성 또는 최선책, 기본 상태를 지칭합니다. 다음과 같은 완화 요소가 현재 상황에서 유용할 수 있습니다.
  
-   기본 구성의 Windows 2000 서비스 팩 4, Windows XP 서비스 팩 2, Windows Server 2003 서비스 팩 1 및 Windows Server 2003 서비스 팩 2에는 UNIX용 Windows 서비스 3.0 및 UNIX용 Windows 서비스 3.5이 포함되어 잇지 않습니다. UNIX용 Windows 서비스 3.0 및 UNIX용 Windows 서비스 3.5는 Windows 2000 서비스 팩 4, Windows XP 서비스 팩 2, Windows Server 2003 서비스 팩 1 및 Windows Server 2003 서비스 팩 2에 옵션으로 설치될 수 있습니다. Windows Vista 및 Windows Server 2003은 UNIX 기반 응용 프로그램용 하위 시스템이 기본적으로 사용하도록 설정되어 있지 않습니다. UNIX 기반 응용 프로그램용 하위 시스템은 Windows Vista 및 Windows Server 2003의 경우 옵션 Windows 구성 요소입니다.
  
#### UNIX용 Windows 서비스 권한 상승 문제점 - CVE-2007-3036에 대한 대안
  
이 취약점에 대한 대안을 확인하지 못했습니다.
  
#### UNIX용 Windows 서비스 권한 상승 문제점 - CVE-2007-3036에 대한 FAQ
  
**이 취약점을 악용하면 어느 작업까지 가능합니까? **  
UNIX용 Windows 서비스 3.0, UNIX용 Windows 서비스 3.5 및 UNIX 기반 응용 프로그램용 하위 시스템에 권한 상승 취약점이 존재하며, 특정 setuid 바이너리 파일을 실행하는 경우 공격자 권한 상승이 발생할 수 있습니다. 이 취약점 악용에 성공한 공격자는 권한을 상승시킬 수 있습니다. 공격자는 프로그램을 설치할 수 있을 뿐 아니라 데이터를 보거나 변경하거나 삭제할 수 있습니다.
  
**취약점의 원인은 무엇입니까? **  
UNIX용 Windows 서비스 3.0, UNIX용 Windows 서비스 3.5 및 UNIX 기반 응용 프로그램용 하위 시스템이 setuid 바이너리 파일을 부정확하게 처리합니다.
  
**공격자는 취약점을 악용하여 무엇을 할 수 있습니까? **  
이 취약점 악용에 성공한 공격자는 영향을 받는 시스템에서 권한이 상승될 수 있습니다. 게스트 운영 체제에 대해 사용자 권한이 적게 구성된 사용자는 게스트 운영 체제에 대해 관리자 권한으로 작업하는 사용자에 비해 영향을 적게 받지****않습니다.
  
**공격자는 이러한 취약점을 어떻게 악용합니까? **  
공격자는 영향을 받는 시스템에 로컬로 로그온하여 특정 setuid 바이너리 파일을 실행합니다. 또는 특정 setuid 바이너리 파일을 실행하도록 공격자가 사용자를 유도합니다.
  
**setuid란 무엇입니까? **  
클라이언트 컴퓨터 사용자는 파일에 대해 setuid(set-user-identifier-on-execution) 비트를 설정할 수 있습니다. setuid 비트가 설정된 실행 파일은 해당 파일 소유자의 사용자 ID에서만 실행되고 해당 파일을 실행하려는 사용자의 사용자 ID로는 실행되지 않습니다.
  
**취약점으로 인해 주로 공격받을 위험이 있는 시스템은 무엇입니까? **  
UNIX용 Windows Services 3.0 또는 UNIX용 Windows 서비스 3.5가 설치되어 있는 Windows 2000 서비스 팩 4, Windows XP 서비스 팩 2, Windows Server 2003 서비스 팩 1 및 Windows Server 2003 서비스 팩이 해당됩니다. Windows Server 2003 R2는 Windows Server 2003의 확장판입니다. UNIX 기반 응용 프로그램용 하위 시스템을 사용하도록 설정한 Windows Vista가 해당됩니다.
  
**업데이트는 어떤 기능을 수행합니까? **  
업데이트는 setuid 바이너리 파일에 대한 연결 자격 증명을 정확하게 처리하여 취약점을 제거합니다.
  
**이 보안 공지를 게시했을 때 이 취약점이 공개되어 있었습니까? **  
예. 이 취약점은 공개되었으며, 제한적으로 배포되었습니다.

SMS 2003 ITMU에 대한 자세한 내용은 다음 [Microsoft 웹 사이트 (영문)](http://technet.microsoft.com/ko-kr/sms/bb676783.aspx)를 방문하십시오. SMS 2003은 Microsoft Office Inventory Tool을 사용하여 Microsoft Office 응용 프로그램에 필요한 업데이트를 검색할 수도 있습니다.
  
**Windows** **Vista참고 사항:** Microsoft Systems Management Server 2003 서비스 팩 3은 Windows Vista 관리 기능 지원을 포함합니다.
  
**ITMU 참고 사항:** Windows Server 2003의 UNIX 응용 프로그램용 하위 시스템은 Windows 운영 체제의 일부분이고 MBSA 2.0.1은 이 구성 요소 검색을 지원합니다. SMS 2003은 ITMU를 사용하여 이 업데이트를 검색하고 배포할 수 있습니다.
  
**EST 참고 사항:** Unix 3.0용 서비스 및 Unix 3.5용 서비스는 Windows 2000, Windows XP, Windows Vista 및 Windows Server 2003을 위한 완전한 기능의 추가 구성 요소입니다. MBSA 2.0.1은 이러한 응용 프로그램의 업데이트 검색을 지원하지 않습니다. [2007년 9월 Enterprise Scan Tool(EST) (영문)](http://www.microsoft.com/downloads/details.aspx?familyid=d7e0af00-47b7-46d0-9252-cc9d7db7385d)은 지원되는 모든 플랫폼에서 MS07-053 필수 업데이트를 검색하기 위해 제공됩니다. 업데이트 배포는 SMS에서 표준 소프트웨어 배포로 처리할 수 있습니다.
  
SMS 2.0 및 SMS 2003용 [Extended Security Update Inventory Tool (영문)](http://www.microsoft.com/downloads/details.aspx?familyid=2c93da1d-48a0-4e5c-991f-87e08954f61b)은 EST와 동일한 엔진을 사용하므로 MBSA에서 지원하는 않는 제품용 업데이트를 SMS에서 검색할 때 사용할 수 있습니다. 이 도구의 최신 버전은 MBSA에서 검색할 수 없는 MS04-028 이후 업데이트의 누적 업데이트를 제공합니다. Extended Security Update Inventory Tool 이전 버전이 SMS에 설치되어 있는 경우 최신 버전으로 업그레이드해야만 MS07-053을 검색할 수 있습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 검색, 탐지 도구 및 지침 섹션을 참조하십시오.

-   **파일 버전 확인**

Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

1.  **시작**, **검색**을 차례로 클릭합니다.
2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
    **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
    **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

#### 배포 정보

**업데이트 설치**

이 보안 업데이트를 설치할 때 설치 관리자는 해당 컴퓨터에서 하나 이상의 업데이트할 파일이 Microsoft 핫픽스에 의해 이미 업데이트되었는지 확인합니다.

설치 프로그램에 대한 자세한 내용은 [Microsoft TechNet 웹 사이트 (영문)](http://go.microsoft.com/fwlink/?linkid=38951)를 방문하십시오.

*핫픽스*와 같이 이 공지에서 사용하는 용어에 대한 자세한 내용은 [Microsoft 기술 자료 문서 824684](http://support.microsoft.com/kb/824684)를 참조하십시오.

이 보안 업데이트는 다음 설치 스위치를 지원합니다.

| 전환    | 설명                                                                                                   |
|---------|--------------------------------------------------------------------------------------------------------|
| **/y**  | 제거를 수행합니다(/m이나 /q를 사용하는 경우에만 해당).                                                 |
| **/f**  | 시스템을 종료할 때 프로그램을 강제로 종료합니다.                                                       |
| **/n**  | Uninstall 폴더를 만들지 않습니다.                                                                      |
| **/z:** | 업데이트 완료 후 다시 시작하지 않습니다.                                                               |
| **/q**  | 사용자 인터페이스를 사용하지 않고 자동 또는 무인 설치 모드를 사용합니다(이 스위치는 /m의 상위 집합임). |
| **/m**  | 사용자 작업이 있는 무인 설치 모드를 사용합니다.                                                        |
| **/l**  | 설치된 핫픽스 목록을 표시합니다.                                                                       |

**참고** 한 명령에서 이러한 스위치를 조합하여 사용할 수 있습니다. 보안 업데이트에서는 이전 버전과의 호환성을 위해 이전 버전의 설치 프로그램에서 사용하는 설치 스위치도 지원합니다. 지원되는 설치 스위치에 대한 자세한 내용은 [Microsoft 기술 자료 문서 262841](http://support.microsoft.com/kb/262841)을 참조하십시오.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 검색, 탐지 도구 및 지침 섹션을 참조하십시오.

-   **파일 버전 확인**

Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

1.  **시작**, **검색**을 차례로 클릭합니다.
2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
    **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
    **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.
