Microsoft 보안 공지 MS10-041 - 중요
===================================

Microsoft .NET Framework의 취약점으로 인한 변조 문제점(981343)
--------------------------------------------------------------

게시된 날짜: 2010년 6월 9일 수요일 | 업데이트된 날짜: 2010년 6월 23일 수요일

**버전:** 1.3

### 일반 정보

#### 요약

이 보안 업데이트는 Microsoft .NET Framework의 공개된 취약점을 해결합니다. 이 취약점을 악용하면 서명된 XML 콘텐츠의 데이터를 탐지되지 않는 방식으로 변조할 수 있습니다. 사용자 지정 응용 프로그램에서 보안 영향은 관련 응용 프로그램이 서명된 콘텐츠를 사용하는 방식에 따라 달라집니다. 서명된 XML 메시지가 보안 채널(예: SSL)을 통해 전송되는 시나리오는 이 취약점의 영향을 받지 않습니다.

이 보안 업데이트의 심각도는 영향을 받는 모든 Microsoft Windows 2000, Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008, Windows 7 및 Windows Server 2008 R2용 Microsoft .NET Framework 릴리스에 대해 중요입니다. 자세한 내용은 이 섹션에서 **영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어**를 참조하십시오.

이 보안 업데이트는 Microsoft .NET Framework에서 XMLDsig 권고 사항이 구현되는 방식을 변경하여 취약점을 해결합니다. 취약점에 대한 자세한 내용은 다음 섹션**,취약점 정보**에서 각 취약점 항목의 자주 제기되는 질문 사항(FAQ)을 참조하십시오.

**권장 사항.**  대다수의 고객은 자동 업데이트를 사용하고 있기 때문에 따로 조치를 취할 필요가 없습니다. 이 보안 업데이트가 자동으로 다운로드되고 설치됩니다. 자동 업데이트를 사용하고 있지 않은 고객은 수동으로 업데이트를 확인하고 이 업데이트를 설치해야 합니다. 자동 업데이트의 특정 구성 옵션에 대한 자세한 내용은 [Microsoft 기술 자료 문서 294871](http://support.microsoft.com/kb/294871)을 참조하십시오.

관리자 및 기업 설치의 경우나 이 보안 업데이트를 수동으로 설치하려는 최종 사용자의 경우에는 고객이 업데이트 관리 소프트웨어를 사용하거나 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 서비스를 통해 업데이트를 확인하여 업데이트를 최대한 빨리 적용하는 것이 좋습니다.

이 공지 뒷부분에 있는 **검색, 탐지 도구 및 지침** 섹션도 참조하십시오.

**문제점.** 이 보안 업데이트를 설치할 때 고객이 겪을 수 있는 현재까지 알려진 문제점은 [Microsoft 기술 자료 문서 981343](http://support.microsoft.com/kb/981343)에 나와 있습니다. 이 문서는 이러한 문제점에 대한 권장 해결 방법도 소개합니다.

#### 영향을 받는 소프트웨어 및 영향을 받지 않는 소프트웨어

다음 소프트웨어는 테스트를 거쳐 영향을 받는 버전 또는 에디션이 확인되었습니다. 다른 버전은 지원 기간이 끝났거나 영향을 받지 않습니다. 사용 중인 소프트웨어 버전에 대한 지원 기간을 확인하려면 [Microsoft 지원 기간 정책 웹 사이트](http://go.microsoft.com/fwlink/?linkid=21742)를 참조하십시오.

이 보안 업데이트와 관련된 자주 제기되는 질문 사항
-------------------------------------------------


**이 공지가 2010년 6월 23일에 개정된 이유는 무엇입니까?**  
Microsoft는 Microsoft .NET Framework 3.5 서비스 팩 1이 설치된 시스템에서 2010년 6월 8일의 Microsoft .NET Framework 3.5(KB982865)용 보안 업데이트가 MBSA(Microsoft Baseline Security Analyzer)에 의해 "설치됨"으로 잘못 나열되었던 문제를 해결하는 검색 로직 변경 사항을 알리기 위해 이 공지를 개정하였습니다. 그렇지만 Microsoft .NET Framework 2.0 서비스 팩 2 및 Microsoft .NET Framework 3.5 서비스 팩 1(KB979909)용 보안 업데이트의 적용은 MBSA에서 올바르게 검색됩니다. 이 변경 사항은 최초 업데이트에 포함된 파일에 영향을 주지 않는 검색 변경 사항에 해당합니다. 시스템을 성공적으로 업데이트한 고객은 이 업데이트를 다시 설치할 필요가 없습니다.

이 검색 변경 사항으로 인해 Windows Server Update Services(WSUS)를 사용하는 고객에게는 KB982865 업데이트가 새 업데이트로 표시될 수 있습니다. 자동 승인 규칙이 없는 경우 업데이트를 수동으로 재승인해야 할 수 있습니다. 업데이트를 재승인하지 않으면 업데이트가 적용되지 않는 경우에도 시스템에 업데이트가 계속 제공될 수 있습니다. 이러한 검색 변경 사항에 대한 자세한 내용은 [Microsoft 기술 자료 문서 894199](http://support.microsoft.com/kb/894199)를 참조하십시오.

**데이터 변조 취약점이란 무엇입니까?**  
정보 보안의 측면에서 데이터 변조 취약점을 악용하면 데이터의 악의적인 수정이 가능합니다. 데이터베이스에 유지되는 영구적 데이터를 무단으로 변경하거나, 인터넷 같은 공개 네트워크를 통해 두 컴퓨터 사이에서 전달되는 데이터를 바꾸는 행위가 여기에 포함됩니다. 이 취약점과 관련하여, 데이터 디지털 서명에 효과적인 암호화 서명을 사용하고 있는 경우 서명된 후에 데이터가 수정되어 디지털 서명 확인에 실패하면 변조를 나타냅니다. 데이터가 변조되었음에도 불구하고 서명 확인이 성공한 경우에도 변조를 나타냅니다.

**설치되어 있는 Microsoft .NET Framework의 버전은 어떻게 알 수 있습니까?**  
시스템에 여러 버전의 .NET Framework를 동시에 설치하고 실행할 수 있으며 순서에 관계 없이 버전을 설치할 수 있습니다. 현재 설치되어 있는 .NET Framework의 버전을 확인하는 방법은 여러가지입니다. 자세한 내용은 [Microsoft 기술 자료 문서 318785](http://support.microsoft.com/kb/318785) 또는 MSDN 문서, [설치된 .NET Framework 버전 확인 (영문)](http://msdn.microsoft.com/en-us/library/y549e41e.aspx)을 참조하십시오.

**지원되는 일부 운영 체제에서 Microsoft .NET Framework 3.5 및 Microsoft .NET Framework 3.5 서비스 팩 1이 영향을 받는 이유는 무엇입니까?**  
Microsoft .NET Framework 3.5에는 Microsoft .NET Framework 2.0 서비스 팩 1 및 Microsoft .NET Framework 3.0 서비스 팩 1 하위 구성 요소가 포함되어 있습니다. Microsoft .NET Framework 3.5 서비스 팩 1에는 Microsoft .NET Framework 2.0 서비스 팩 2 및 Microsoft .NET Framework 3.0 서비스 팩 2 하위 구성 요소가 포함되어 있습니다. 일부 운영 체제의 경우 Microsoft .NET Framework 3.5 및 Microsoft .NET Framework 3.5 서비스 팩 1에서 이러한 취약한 하위 구성 요소를 설치할 수도 있습니다.

**이 공지에 나열되어 있지 않은 버전의 Microsoft .NET Framework가 시스템에 설치되어 있습니다. 이 구성이 이 취약점의 영향을 받습니까?**  
이 공지에 나열된 영향을 받는 소프트웨어는 테스트를 거쳐 해당 버전이 영향을 받는지 확인되었습니다. 다른 버전은 지원 기간이 끝나거나 지원되지 않습니다. 시스템에 지원되지 않는 버전의 Microsoft .NET Framework를 설치한 고객은 해당 버전의 Microsoft .NET Framework를 제거한 후 최신 버전의 Microsoft .NET Framework로 업그레이드하는 것이 좋습니다.

Microsoft .NET Framework는 제어판의 **프로그램 추가/제거** 도구를 사용하여 제거할 수 있습니다. 특정 버전의 Microsoft .NET Framework를 제거하는 것에 대한 자세한 내용은 [Microsoft 기술 자료 문서 320122](http://support.microsoft.com/kb/320112), [Microsoft 기술 자료 문서 824643](http://support.microsoft.com/kb/824643) 및 [Microsoft 기술 자료 문서 908077](http://support.microsoft.com/kb/908077)을 참조하십시오.

**이 업데이트에 기능에 대한 보안 관련 변경 사항이 포함되어 있습니까?**  
예. 이 공지의 **취약점 정보**에 나열된 변경 사항 외에, 이 업데이트에는 ASP.NET 요청 유효성 확인 기능에 대한 심층 보안 변경 사항이 포함되어 있습니다. 이 변경 사항은 공격자가 ASP.NET 기반 웹 사이트에서 기본적으로 사용되는 기본 심층 보안 대응책을 우회할 수 있는 문제점을 해결합니다.

이 문제점은 ASP.NET의 요청 유효성 확인 기능이 특정 문자 시퀀스를 적절하게 확인하지 못하기 때문에 발생합니다. ASP.NET 요청 유효성 확인 기능은 신뢰할 수 없는 입력 변수를 제한하는 효과적인 유효성 확인 계층을 대체할 수 없습니다. ASP.NET에서 웹 응용 프로그램에 제공하는 보안 기능에 대해 자세히 알고자 하는 개발자는 MSDN 문서, [ASP.NET 기본 제공 기능을 활용하여 웹 공격 차단 (영문)](http://msdn2.microsoft.com/en-us/library/ms972969.aspx)을 참조하십시오.

웹 사이트에 신뢰할 수 없는 사용자 입력을 제한하는 효과적인 유효성 확인 계층이 없는 경우 이 취약점 악용에 성공한 공격자는 영향을 받는 웹 사이트에 ASP.NET 콘텐츠를 비롯한 임의의 콘텐츠를 삽입할 수 있습니다. 이 심층 보안 변경 사항은 이 비공개적으로 보고된 문제점을 완화합니다.

**ASP.NET 요청 유효성 확인이란 무엇입니까?**  
ASP.NET 요청 유효성 확인 기능에 대한 이 심층 보안 변경 사항은 ASP.NET을 실행하는 웹 사이트에서 기본 입력 유효성 확인을 수행합니다. 하지만 ASP.NET 요청 유효성 확인 기능은 신뢰할 수 없는 입력 변수를 제한하는 효과적인 유효성 확인 계층을 대체할 수 없습니다. ASP.NET에서 웹 응용 프로그램에 제공하는 보안 기능에 대해 자세히 알고자 하는 개발자는 MSDN 문서, [ASP.NET 기본 제공 기능을 활용하여 웹 공격 차단 (영문)](http://msdn2.microsoft.com/en-us/library/ms972969.aspx)및 [ASP.NET 요청 유효성에 의존하지 않음 (영문)](http://msdn.microsoft.com/en-us/library/ms998264.aspx)을 참조하십시오.

**ASP.NET 웹 응용 프로그램이 이 심층 보안 변경 사항의 영향을 받는지 어떻게 확인할 수 있습니까?**  
모든 신뢰할 수 없는 입력 변수를 예상된 값 또는 문자 범위로 제한하는 ASP.NET 개발 웹 응용 프로그램은 영향을 받지 않습니다. ASP.NET 웹 응용 프로그램 보안 강화에 대한 자세한 내용은 [Microsoft 기술 자료 문서 815155](http://support.microsoft.com/kb/815155)를 참조하십시오.

**ASP.NET이란 무엇입니까?**  
ASP.NET은 개발자가 웹 응용 프로그램과 XML 웹 서비스를 구축하는 데 사용하는 Microsoft .NET Framework 내의 기술 집합입니다.

정적 HTML과 스크립팅을 사용하는 기존 웹 페이지와 달리, ASP.NET은 컴파일된 이벤트 구동 페이지를 사용합니다. ASP.NET이 웹 기반 응용 프로그램 환경이므로 기본적인 HTTP 기능성을 제공하기 위해서 기본 웹 서버가 필요하며, ASP.NET은 IIS(인터넷 정보 서비스)에서 실행됩니다. 자세한 내용은 [The Official Microsoft ASP.NET 사이트 (영문)](http://www.asp.net/)를 참조하십시오.

**이 문제점을 심층 보안 대응책으로 해결하는 이유는 무엇입니까?**  
ASP.NET 요청 유효성 확인은 개발자가 제공하는 입력 유효성 확인과 더불어 추가적인 예방 수단으로 사용할 수 있도록 심층 보안 변경 사항으로 처리됩니다. 개발자만이 특정 응용 프로그램에서 올바른 입력을 구성하는 내용을 정의할 수 있습니다. 여러 개의 심층 보안 계층이 존재하면 공격자가 관련 시스템의 보안을 침해하지 못하도록 방지하는 데 큰 도움이 되지만, 심층 보안 기능은 완전한 방어를 제공하도록 설계된 것이 아닙니다. 따라서 ASP.NET 요청 유효성 확인에서 특정 문자 시퀀스가 누락된 경우에도 이 문제점의 최대 보안 영향이 이 기능을 모두 사용하지 않는 경우보다 클 수 없습니다.

**심층 보안이란 무엇입니까?**  
정보 보안에서, 심층 보안이란 공격자가 네트워크 또는 시스템의 보안을 침해하지 못하게 하기 위해 여러 계층의 방어 기전을 배치하는 접근 방식을 의미합니다.

**파일 정보 세부 사항은 어디에 있습니까?**  
파일 정보 세부 사항의 위치에 대해서는 **보안 업데이트 배포** 섹션에서 참조 표를 참조하십시오.

XML 서명 HMAC 잘림 인증 우회 취약점(CVE-2009-0217)
--------------------------------------------------


Microsoft .NET Framework에 공격자가 탐지되지 않은 방식으로 서명된 XML 콘텐츠를 변조할 수 있는 데이터 변조 취약점이 존재합니다. 사용자 지정 응용 프로그램에서 보안 영향은 관련 사용 시나리오에 따라 달라집니다. 서명된 XML 메시지가 보안 채널(예: SSL)을 통해 전송되는 시나리오는 이 취약점의 영향을 받지 않습니다.

이 취약점을 CVE(Common Vulnerabilities and Exposures) 목록의 표준 항목으로 보려면 [CVE-2009-0217 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0217)을 참조하십시오.

#### XML 서명 HMAC 잘림 인증 우회 취약점(CVE-2009-0217)에 대한 완화 요소

완화란 취약점의 악용 심각도를 낮출 수 있는 설정, 일반적인 구성 또는 최선책, 기본 상태를 지칭합니다. 다음과 같은 완화 요소가 현재 상황에서 유용할 수 있습니다.

-   Microsoft Office는 이 데이터 변조 취약점을 악용하는 공격자가 침해할 수 없는 방식으로 XML 서명 확인을 구현합니다.

#### XML 서명 HMAC 잘림 인증 우회 취약점(CVE-2009-0217)에 대한 대안

대안이란 기본적인 취약점을 수정하지는 않지만 업데이트를 적용하기 전에 알려진 공격 경로를 차단하는 데 유용한 설정 또는 구성 변경을 지칭합니다. Microsoft는 다음과 같은 대안을 테스트했으며, 대안으로 인한 기능 저하 여부를 해당 항목에서 설명합니다.

-   **보안 채널을 사용하여 중요한 정보 전송**

    응용 프로그램이 민감한 XML 데이터나 응용 프로그램의 보안에 중요한 XML 데이터를 전송한다면 보안 채널을 사용하여 해당 정보를 전송하십시오. 예를 들어 IIS(인터넷 정보 서비스) 서버를 사용한다면 민감한 응용 프로그램이나 보안에 중요한 응용 프로그램에 SSL을 사용하도록 요구합니다.

#### XML 서명 HMAC 잘림 인증 우회 취약점(CVE-2009-0217)에 대한 FAQ

**이 취약점을 악용하면 어느 작업까지 가능합니까?**  
이 취약점은 암호화 서명을 우회하는 데 사용할 수 있는 데이터 변조 취약점입니다.

**취약점의 원인은 무엇입니까?**  
이 취약점은 Microsoft .NET Framework에서 W3C XMLDsig(XML Signature Syntax and Processing) 권고 사항을 구현하는 방식에서 발생합니다. XMLDsig에 대한 자세한 내용은 W3C Recommendation, [XML Signature Syntax and Processing (Second Edition) (영문)](http://www.w3.org/tr/xmldsig-core/)을 참조하거나 [Errata for XML Signature Second Edition (영문)](http://www.w3.org/2008/06/xmldsigcore-errata.html)의 "E03 HMAC truncation (CVE-2009-0217)"을 참조하십시오.

**Microsoft .NET Framework란 무엇입니까?**  
Microsoft .NET Framework는 Microsoft Windows 운영 체제의 구성 요소로, 이를 이용하여 소프트웨어 응용 프로그램과 웹 서비스를 작성하고 실행할 수 있습니다. 여기에는 웹 서비스와 웹 응용 프로그램(ASP.NET), 데이터 액세스(ADO.NET), 스마트 클라이언트 응용 프로그램(Windows Forms) 등과 같은 많은 기술이 포함됩니다.

**HMAC란 무엇입니까?**  
HMAC(Hash-based Message Authentication Code)는 보낸 사람과 받는 사람이 비밀 키를 공유하는 경우 메시지의 무결성과 진위 여부를 확인하여 비보안 채널을 통해 전송된 메시지가 변조되지 않았는지 확인하는 데 사용할 수 있습니다.

**XML이란 무엇입니까?**  
XML(Extensible Markup Language)은 구조적 데이터를 기술하는 형식을 제공하는 마크업 언어입니다. XML(Extensible Markup Language)은 W3C(World Wide Web Consortium) 사양이며 SGML(Standard Generalized Markup Language) 하위 집합입니다. XML에 대한 자세한 내용은 MSDN 데이터 개발자 센터에서 [XML 홈 페이지 (영문)](http://msdn.microsoft.com/xml/)를 참조하거나 W3C Recommendation, [Extensible Markup Language (XML) 1.0 (Fifth Edition) (영문)](http://www.w3.org/tr/rec-xml/)을 참조하십시오.

**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**  
취약점 악용에 성공한 공격자는 특정 암호화 서명을 우회할 수 있으며 받는 사람이 변경 사항을 탐지하지 못하는 방식으로 서명된 XML 콘텐츠를 변조할 수 있습니다. 메시지를 원래 의미와 완전히 다르게 변경할 수 있다면 스푸핑에 이용될 수 있습니다. 개발자가 HMAC 서명 XML 콘텐츠에 의존하는 응용 프로그램을 개발한 경우 보안 영향은 해당 응용 프로그램의 관련 사용 시나리오에 따라 달라집니다.

**공격자는 이러한 취약점을 어떻게 악용합니까?**  
공격자는 특수하게 조작된 XML 콘텐츠를 취약한 시스템으로 보내야 합니다. Microsoft Office는 공격자가 악용할 수 없는 방식으로 XML 서명 확인을 구현합니다.

**취약점으로 인해 주로 공격받을 위험이 있는 시스템은 무엇입니까?**  
암호화 작업을 위해 HMAC(Hash-based Message Authentication Code)에 의존하는 시스템이나 응용 프로그램이 주로 이 취약점으로 인한 위험에 노출됩니다. Microsoft Office는 이 취약점을 악용하는 공격자가 침해할 수 없는 방식으로 XML 서명 확인을 구현합니다.

**업데이트는 어떤 기능을 수행합니까?**  
이 업데이트는 Microsoft .NET Framework에서 XMLDsig 권고 사항을 구현하는 방식을 수정합니다.

**이 보안 공지를 게시했을 때 이 취약점이 공개되어 있었습니까?**  
예. 이 취약점은 공개되었습니다. 이 취약점에는 CVE(Common Vulnerability and Exposure) 번호 [CVE-2009-0217 (영문)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0217)이 지정되었습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 검색, 탐지 도구 및 지침 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음 **속성**을 클릭합니다.
        **참고** 운영 체제 버전 또는 시스템에 설치된 프로그램에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 시스템에 설치된 파일 버전을 확인합니다.
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리 키 확인**

    이 섹션의 **참조 표**에 나열된 레지스트리 키를 검토하여 이 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    이 레지스트리 키에 설치된 파일의 전체 목록 중 일부가 들어 있지 않을 수 있습니다. 또한 관리자나 OEM이 보안 업데이트를 Windows 설치 원본 파일에 통합하거나 포함시키는 경우 이러한 레지스트리 키가 제대로 만들어지지 않을 수 있습니다.

**Microsoft .NET Framework 2.0 서비스 팩 1 및 Microsoft .NET Framework 3.5(KB979913), Microsoft .NET Framework 2.0 서비스 팩 2 및 Microsoft .NET Framework 3.5 서비스 팩 1(KB 979910 및 KB979911)용 업데이트 설치**

이 보안 업데이트를 설치할 때 설치 관리자는 해당 컴퓨터에서 하나 이상의 업데이트할 파일이 Microsoft 핫픽스에 의해 이미 업데이트되었는지 확인합니다.

*핫픽스*와 같이 이 공지에서 사용하는 용어에 대한 자세한 내용은 [Microsoft 기술 자료 문서 824684](http://support.microsoft.com/kb/824684)를 참조하십시오.

이 보안 업데이트는 다음 설치 스위치를 지원합니다.

지원되는 보안 업데이트 설치 스위치 

| 스위치            | 설명                                                                                                               |
|-------------------|--------------------------------------------------------------------------------------------------------------------|
| **/?, /h, /help** | 지원되는 스위치에서 도움말을 표시합니다.                                                                           |
| **/quiet**        | 상태 표시 또는 오류 메시지를 표시하지 않습니다.                                                                    |
| **/norestart**    | **/quiet**과 함께 사용하면 설치 완료를 위해 다시 시작이 필수적인 경우에도 시스템이 설치 후 다시 시작되지 않습니다. |

**참고** wusa.exe 설치 프로그램에 대한 자세한 내용은 [Microsoft 기술 자료 문서 934307](http://support.microsoft.com/kb/934307)을 참조하십시오.

**적용된 업데이트 확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**을 클릭한 다음 **검색 시작**에 업데이트 파일 이름을 입력합니다.
    2.  **프로그램**에 해당 파일이 나타나면 파일 이름을 마우스 오른쪽 단추로 클릭하고 **속성**을 클릭합니다.
    3.  **일반** 탭에 표시된 파일 크기를 공지 KB 문서에 제공된 파일 정보 표와 비교합니다.
    4.  **세부 정보** 탭 및 비교 정보를 클릭하여 파일 버전과 수정된 날짜를 공지 KB 문서에 제공된 파일 정보 표와 비교할 수도 있습니다.
    5.  마지막으로, **이전 버전** 탭을 클릭하여 표시된 이전 파일 버전의 파일 정보를 새 파일 또는 업데이트된 파일 버전의 파일 정보와 비교할 수도 있습니다.

-   **파일 버전 확인**

    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

#### 배포 정보
  
**업데이트 설치**
  
이 보안 업데이트를 설치할 때 설치 관리자는 해당 컴퓨터에서 하나 이상의 업데이트할 파일이 Microsoft 핫픽스에 의해 이미 업데이트되었는지 확인합니다.
  
*핫픽스*와 같이 이 공지에서 사용하는 용어에 대한 자세한 내용은 [Microsoft 기술 자료 문서 824684](http://support.microsoft.com/kb/824684)를 참조하십시오.
  
이 보안 업데이트는 다음 설치 스위치를 지원합니다.
  
지원되는 보안 업데이트 설치 스위치 

| 스위치            | 설명                                                                                                               |  
|-------------------|--------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | 지원되는 스위치에서 도움말을 표시합니다.                                                                           |  
| **/quiet**        | 상태 표시 또는 오류 메시지를 표시하지 않습니다.                                                                    |  
| **/norestart**    | **/quiet**과 함께 사용하면 설치 완료를 위해 다시 시작이 필수적인 경우에도 시스템이 설치 후 다시 시작되지 않습니다. |
  
**참고** wusa.exe 설치 프로그램에 대한 자세한 내용은 [Microsoft 기술 자료 문서 934307](http://support.microsoft.com/kb/934307)을 참조하십시오.
  
**적용된 업데이트 확인**
  
-   **Microsoft Baseline Security Analyzer**
  
    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 자세한 내용은 이 공지 앞부분에 있는 **검색, 탐지 도구 및 지침** 섹션을 참조하십시오.
  
-   **파일 버전 확인**
  
    Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.
  
    1.  **시작**을 클릭한 다음 **검색 시작**에 업데이트 파일 이름을 입력합니다.  
    2.  **프로그램**에 해당 파일이 나타나면 파일 이름을 마우스 오른쪽 단추로 클릭하고 **속성**을 클릭합니다.  
    3.  **일반** 탭에 표시된 파일 크기를 공지 KB 문서에 제공된 파일 정보 표와 비교합니다.  
    4.  **세부 정보** 탭 및 비교 정보를 클릭하여 파일 버전과 수정된 날짜를 공지 KB 문서에 제공된 파일 정보 표와 비교할 수도 있습니다.  
    5.  마지막으로, **이전 버전** 탭을 클릭하여 표시된 이전 파일 버전의 파일 정보를 새 파일 또는 업데이트된 파일 버전의 파일 정보와 비교할 수도 있습니다.
