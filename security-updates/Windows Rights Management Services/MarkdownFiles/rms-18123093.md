RMS 등록
========

서버 등록 프로세스에서는 서버 사용 허가자 인증서를 만들고 전달합니다. 서버 사용 허가자 인증서는 배포 중인 서버 ID의 유효성을 검사하고 RMS로 보호된 콘텐츠 사용 도중 자격 증명 유효성 검사를 제공합니다. 모든 라이센스 클러스터의 첫 번째 서버는 구축 프로세스의 일부로 루트 인증 클러스터에 등록됩니다. 클러스터에 있는 후속 서버는 별도로 등록되지 않습니다.

루트 인증 클러스터의 첫 번째 서버(루트 인증 서버)는 Microsoft 등록 서비스에 등록해야 합니다. 이 프로세스는 루트 인증 서버가 인터넷에 연결되어 있는 경우 구축의 일부로 자동으로 수행될 수 있습니다. 또는 요청을 파일로 내보낸 다음 이 파일을 인터넷에 연결된 다른 컴퓨터에서 Microsoft 등록 서비스로 전송하여 등록 프로세스를 완료할 수도 있습니다. 등록 요청은 RMS 관리 웹 페이지를 사용하여 가져올 수 있는 루트 인증 서버용 서버 사용 허가자 인증서를 반환합니다.

등록 요청에는 다음 정보가 포함됩니다.

-   해지 정보. RMS 설치가 표준 또는 사용자 지정(타사) 해지를 사용하는지 여부 타사 해지를 사용 중인 경우 해지 기관의 공개 키가 포함됩니다.
-   인증서 공개 키. 서버 사용 허가자 인증서의 공개 키. 이 공개 키는 RMS 서버에서 생성되며 서버 사용 허가자 인증서를 얻기 위해 Microsoft 서버 등록 서비스로 보내집니다.
-   SKU. 공식 RMS SKU 제목.
-   버전. RMS 어셈블리 버전 번호.
-   URL. RMS 서버 클러스터 기반 URL.

Microsoft 서버 등록 서비스에서 등록 요청에 응답을 제공하게 되면 다음 정보가 XML 형식으로 RMS 서버에 반환됩니다.

-   서버 사용 허가자 인증서.
-   서명 기관의 인증서 체인.

RMS 루트 인증 서버가 온라인 또는 오프라인 방법을 사용하여 등록되었는지 여부에 상관없이 같은 정보가 전송됩니다. 어느 방법을 사용하든 추가 정보는 수집되지 않습니다.

오프라인 서버 등록을 수행하는 단계에 대한 자세한 내용은 이 설명서 모음의 “RMS 서버 작동”에 나오는 "오프라인 등록을 사용하여 루트 인증 서버 등록"을 참조하십시오.

클라이언트 등록 프로세스에서는 작성자가 회사 네트워크에 연결되어 있지 않은 컴퓨터에서 RMS로 보호된 콘텐츠를 게시할 수 있도록 해주는 클라이언트 사용 허가자 인증서를 만들고 전달합니다. 작성자는 언제든지 클라이언트 사용 허가자 인증서를 요청할 수 있습니다. 클라이언트 등록은 필요하지 않습니다.

모든 등록 요청이 로그됩니다.
