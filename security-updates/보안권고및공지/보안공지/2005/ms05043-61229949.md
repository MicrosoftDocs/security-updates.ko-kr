Security Bulletin

Microsoft 보안 공지 MS05-043 - 긴급
===================================

인쇄 스풀러 서비스의 취약점으로 인한 원격 코드 실행 문제점(896423)
------------------------------------------------------------------

게시된 날짜: 2005년 8월 9일 화요일 | 업데이트된 날짜: 2007년 4월 18일 수요일

**버전:** 1.2

#### 요약

**이 내용을 읽어야 할 사람:** Microsoft Windows를 사용하는 고객

**취약점으로 인한 영향:** 원격 코드 실행

**최대 심각도:** 긴급

**권장 사항:** 즉시 업데이트를 적용하십시오.

**보안 업데이트 대체:** 없음

**주의 사항:** 이 보안 업데이트를 설치할 때 고객이 겪을 수 있는 현재까지 알려진 문제점은 [Microsoft 기술 자료 문서 908506](http://support.microsoft.com/kb/908506)에 나와 있습니다. 이 문서는 이러한 문제점에 대한 권장 해결 방법도 소개합니다. 자세한 내용은 [Microsoft 기술 자료 문서 908506](http://support.microsoft.com/kb/908506)을 참조하십시오.

**테스트된 소프트웨어 및 보안 업데이트 다운로드 위치:**

**영향을 받는 소프트웨어:**

-   Microsoft Windows 2000 서비스 팩 4 – [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=3dd3b530-7f43-4c18-8298-6e8797431a5d&displaylang=ko)
-   Microsoft Windows XP 서비스 팩 1 및 Microsoft Windows XP 서비스 팩 2 – [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=ef402946-1c3b-47e9-9d51-77d890df8725&displaylang=ko)
-   Microsoft Windows Server 2003 – [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=25469675-df28-4889-8d13-25efcd498388&displaylang=ko)
-   Microsoft Windows Server 2003 (Itanium 기반 시스템용) – [업데이트 다운로드](http://www.microsoft.com/downloads/details.aspx?familyid=f0aec064-34a3-4ee4-9f15-be1e3dd02bc7)

**영향을 받지 않는 소프트웨어:**

-   Microsoft Windows XP Professional x64 Edition
-   Microsoft Windows Server 2003 서비스 팩 1
-   Microsoft Windows Server 2003 SP1(Itanium 기반 시스템용)
-   Microsoft Windows Server 2003 x64 Edition
-   Microsoft Windows 98, Microsoft Windows 98 Second Edition(SE) 및 Microsoft Windows Millennium Edition(ME)

위에 나열된 소프트웨어는 테스트를 거쳐 해당 버전이 영향을 받는지 확인되었습니다. 그 밖의 다른 버전은 더 이상 보안 업데이트가 지원되지 않거나 영향을 받지 않을 수 있습니다. 사용 중인 제품과 버전에 대한 지원 기간을 확인하려면 [Microsoft 지원 기간 정책 웹 사이트](http://go.microsoft.com/fwlink/?linkid=21742)를 참조하십시오.

취약점 세부 정보
----------------

<span></span>
#### 인쇄 스풀러 취약점 - CAN-2005-1984:

프린터 스풀러 서비스에 원격 코드 실행 취약점이 존재하며, 이 취약점 악용에 성공한 공격자는 영향을 받는 시스템을 완전히 제어할 수 있습니다.

#### 인쇄 스풀러 취약점 - CAN-2005-1984에 대한 완화 요소:

-   Windows XP 서비스 팩 2 및 Windows Server 2003에서 이 취약점은 인증된 사용자로 제한됩니다. 또한 이러한 운영 체제 버전에서 이 문제로 원격 공격 경로를 만들려면 적절한 권한을 가진 로컬 사용자가 먼저 프린터를 공유하거나 공유 프린터에 연결을 시도해야 합니다. 적절한 권한이 없는 사용자가 프린터를 공유하거나 공유 프린터에 연결을 시도하는 경우, 이 취약점을 악용하려면 공격자가 유효한 로그온 자격 증명을 가지고 로컬로 로그온할 수 있어야 합니다.
-   Windows XP 서비스 팩 2 및 Windows Server 2003에서 이 문제는 서비스 거부 상태를 유발합니다. Windows XP 서비스 팩 2 및 Windows Server 2003에서 이 문제점은 원격 코드 실행 또는 권한 상승을 목적으로 악용될 수 없습니다.
    다른 운영 체제 버전에서 이 취약점을 악용하려는 시도는 대부분 서비스 거부 상태를 유발합니다. 그러나 원격 코드 실행은 가능합니다.
-   최선의 방화벽 구성 방법과 표준 기본 방화벽 구성을 이용하면 기업 경계 외부에서 들어오는 공격으로부터 네트워크를 보호할 수 있습니다. 인터넷과 연결되는 시스템의 경우, 필요한 포트만 최소한으로 열어 두는 것이 안전합니다.

#### 인쇄 스풀러 취약점 - CAN-2005-1984에 대한 대안:

Microsoft는 다음과 같은 대안을 테스트했습니다. 이러한 대안이 취약점 자체를 해결하지는 않지만 알려진 공격 경로를 차단하는 데 도움이 됩니다. 대안으로 인해 기능이 저하되는 경우는 아래 섹션에 별도로 표시하였습니다.

-   **인쇄 스풀러 서비스 사용 안 함**

    인쇄 스풀러 서비스를 사용하지 않도록 설정하면 이 취약점을 악용하는 시도로부터 영향을 받는 시스템을 보호할 수 있습니다. 인쇄 스풀러 서비스를 사용하지 않도록 설정하려면 다음 단계를 수행하십시오.

    1.  **시작**, **제어판** 을 차례로 클릭합니다.또는 **설정**을 가리킨 다음 **제어판**을 클릭합니다.
    2.  **관리 도구**를 두 번 클릭합니다.
    3.  **서비스**를 두 번 클릭합니다.
    4.  **인쇄 스풀러**를 두 번 클릭합니다.
    5.  **시작 유형** 목록에서 **사용 안 함**을 클릭합니다.
    6.  **중지**, **확인**을 차례로 클릭합니다.

    명령 프롬프트에서 다음 명령을 사용하여 인쇄 스풀러 서비스를 중지하고 비활성화할 수도 있습니다.

    `sc stop Spooler & sc config Spooler start= disabled`

    **대안의 영향:** 인쇄 스풀러 서비스를 비활성화하는 경우 로컬 또는 원격으로 인쇄할 수 없습니다. 따라서 인쇄가 필요하지 않은 시스템에서만 이 대안을 사용하는 것이 좋습니다.

-   **Windows 2000 Server 서비스 팩 4의 경우 NullSessionPipes 레지스트리 키에서 인쇄 스풀러 서비스를 제거합니다.**
    Windows 2000 Server 서비스 팩 4 및 Windows XP 서비스 팩 1은 영향을 받는 서비스에 대한 익명 연결을 허용합니다. 익명의 공격자가 이 취약점을 악용하지 못하도록 하려면 NullSessionPipes 하위 키에서 인쇄 스풀러 서비스를 제거하십시오. 이 대안은 인증된 사용자의 공격은 차단하지 못합니다. 프린터 스풀러 서비스를 비활성화할 수 없는 경우에만 이 대안을 사용합니다.

    **참고** 레지스트리 편집기를 잘못 사용하면 시스템 전체에 심각한 문제가 발생할 수 있으며 운영 체제를 다시 설치해야 할 수도 있습니다. Microsoft는 레지스트리 편집기를 잘못 사용하여 발생하는 문제에 대한 해결을 보증하지 않습니다. 레지스트리 편집기의 사용에 따른 모든 책임은 사용자에게 있습니다. 레지스트리를 편집하는 방법에 대한 자세한 내용은 레지스트리 편집기(Regedit.exe)의 "키와 값 변경" 도움말 항목 또는 Regedt32.exe의 "레지스트리의 정보 추가와 삭제" 및 "레지스트리 정보 편집" 도움말 항목을 참조하십시오.

    **참고** 레지스트리를 편집하기 전에 백업해 두는 것이 좋습니다.

    1.  **시작**, **실행**을 차례로 클릭하고 "**regedt32**"(따옴표 없이)를 입력한 다음 **확인**을 클릭합니다.
    2.  레지스트리 편집기에서 다음 레지스트리 키를 찾습니다.

        **HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\LanmanServer\\Parameters\\NullSessionPipes**

    3.  레지스트리 키를 편집하여 **SPOOLSS** 값을 제거합니다.
    4.  이 조치를 수행한 다음 영향을 받는 시스템을 다시 시작합니다.

    **대안의 영향:** 인쇄 스풀러 서비스에 대한 익명 연결이 허용되지 않습니다. 이는 이후 운영 체제 버전의 기본 동작입니다.

#### 인쇄 스풀러 취약점 - CAN-2005-1984에 대한 FAQ:

**이 취약점을 악용하면 어느 작업까지 가능합니까?**

이것은 원격 코드 실행 취약점입니다. 이 취약점을 악용한 공격자는 영향을 받는 시스템을 원격으로 완전히 제어할 수 있습니다. 공격자가 프로그램을 설치하거나 데이터를 보고, 변경하거나 삭제하고 모든 사용자 권한이 있는 새 계정을 만들 수도 있습니다. 그러나 이 취약점을 악용하려는 시도는 대부분 서비스 거부 상태를 유발할 수 있습니다.

**취약점의 원인은 무엇입니까?**
인쇄 스풀러 서비스에서 검사되지 않은 버퍼 때문입니다.

**인쇄 스풀러 서비스란 무엇입니까?**
인쇄 스풀러 서비스인 Spoolsv.exe는 서비스로 설치된 실행 파일입니다. 스풀러는 운영 체제를 시작할 때 로드되며 운영 체제를 종료할 때까지 계속 실행됩니다. 인쇄 스풀러 서비스는 올바른 프린터 드라이버의 위치 검색, 해당 드라이버 로드, 높은 수준의 함수 호출을 인쇄 작업에 스풀링, 인쇄 작업 예약 등의 작업을 포함하는 인쇄 프로세스를 관리합니다. 특정 인쇄 작업이 완료되면 인쇄 스풀러 서비스는 인쇄 라우터로 작업을 전달합니다. 인쇄 스풀러 서비스에 대한 자세한 내용은 다음 [웹 사이트](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/techref/d58ce7b9-49cf-4f5e-95e9-1ade005c13e0.mspx)를 방문하십시오.

**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**
원격 코드 실행을 위해 이 취약점 악용에 성공한 공격자는 영향을 받는 시스템을 완전히 제어할 수 있습니다. Windows XP 서비스 팩 2 및 Windows Server 2003에서 이 문제는 서비스 거부 상태를 유발합니다. 다른 운영 체제 버전에서 이 취약점을 악용하려는 시도는 대부분 서비스 거부 상태를 유발합니다. 그러나 원격 코드 실행은 가능합니다.

**누가 취약점을 악용할 수 있습니까?**
Windows 2000 및 Windows XP 서비스 팩 1에서 특수하게 조작한 메시지를 영향을 받는 시스템에 전달할 수 있는 모든 익명 사용자가 이 취약점을 악용할 수 있습니다. Windows XP 서비스 팩 2 및 Windows Server 2003에서 이 취약점은 인증된 사용자로 제한됩니다. 인증된 공격자는 시스템에 로컬로 로그온하고 영향을 받는 모든 운영 체제 버전에서 이 취약점을 악용할 수도 있습니다.

**공격자는 이러한 취약점을 어떻게 악용합니까?**
공격자는 특수하게 조작한 메시지를 조작한 다음, 영향을 받는 시스템에 해당 메시지를 보내는 방법으로 취약점 악용을 원격으로 시도할 수 있습니다. 그러면 메시지는 원격 공격 경로에 취약한 운영 체제 버전과 구성이 있는 영향을 받는 시스템에서 코드를 실행할 수 있습니다. 기본적으로 Windows 2000 및 Windows XP 서비스 팩 1에는 원격 취약점이 있습니다. Windows XP SP2 또는 Windows Server 2003에서는 사용자에게 적절한 권한이 있거나 공유 프린터에 연결을 시도해야만 원격 공격 경로가 만들어집니다.

-   **파일 버전 확인**

    **참고** Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.

        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 경우에 따라 파일 이름이 설치 도중 변경될 수도 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

**Systems Management Server:**

Microsoft Systems Management Server(SMS)는 기업에서 업데이트 관리를 효율적으로 구성할 수 있는 솔루션입니다. 관리자는 SMS를 사용하여 보안 업데이트가 필요한 Windows 기반 시스템을 식별하고, 사용자의 업무 중단을 최소화하면서 기업 전체에 해당 업데이트를 효율적으로 배포할 수 있습니다. 관리자가 SMS 2003을 사용하여 보안 업데이트를 배포할 수 있는 방법에 대한 자세한 내용은 [SMS 2003 보안 패치 관리 웹 사이트](http://www.microsoft.com/korea/smserver/evaluation/capabilities/patch.asp)를 참조하십시오. SMS 2.0 사용자는 [Software Updates Service 기능 팩](http://go.microsoft.com/fwlink/?linkid=33340)을 사용하여 보안 업데이트를 배포할 수도 있습니다. SMS에 대한 자세한 내용은 [SMS 웹 사이트](http://www.microsoft.com/korea/smserver/default.asp)를 참조하십시오.

**참고** SMS는 Microsoft Baseline Security Analyzer, Microsoft Office Detection Tool 및 Enterprise Update Scanning Tool을 사용하여 보안 공지 업데이트 검색 및 배포를 위한 광범위한 지원을 제공합니다. 일부 소프트웨어 업데이트는 이러한 도구로 검색하지 못할 수도 있습니다. 이러한 경우 관리자는 SMS의 인벤토리 기능을 사용하여 특정 시스템을 대상으로 업데이트를 수행할 수 있습니다. 이 절차에 대한 자세한 내용은 다음 [웹 사이트 (영문)](http://go.microsoft.com/fwlink/?linkid=33341)를 참조하십시오. 일부 보안 업데이트 작업을 수행하려면 관리자 권한이 필요하며 컴퓨터를 다시 시작해야 할 수 있습니다. 관리자는 [SMS 2003 Administration Feature Pack (영문)](http://go.microsoft.com/fwlink/?linkid=33387) 및 [SMS 2.0 Administration Feature Pack (영문)](http://go.microsoft.com/fwlink/?linkid=21161)에서 제공되는 Elevated Rights Deployment Tool을 사용하여 이러한 업데이트를 설치할 수 있습니다.
