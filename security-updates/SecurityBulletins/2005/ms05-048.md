#### Collaboration Data Objects 취약점 - CAN-2005-1987
  
Collaboration Data Objects에 원격 코드 실행 취약점이 존재하며, 이 취약점 악용에 성공한 공격자는 영향을 받는 시스템을 완전히 제어할 수 있습니다.
  
#### Collaboration Data Objects 취약점 - CAN-2005-1987의 제한 사항:
  
-   기본적으로 Microsoft 인터넷 정보 서비스 (IIS) 5.0 SMTP (Simple Mail Transfer Protocol) 및 Exchange 2000 Server SMTP 서비스는 Cdosys.dll 파일 및 Codex.dll 파일을 사용하는 이벤트 싱크를 사용하지 않습니다.  
-   기본적으로 IIS 6.0은 Windows Server 2003에서 사용하지 않습니다.  
-   사용하도록 설정할 경우 IIS 6.0은 기본적으로 SMTP를 사용하지 않습니다.
  
#### Collaboration Data Objects 취약점 - CAN-2005-1987에 대한 대안:
  
Microsoft는 다음 대안을 테스트했습니다. 이러한 대안은 취약점 자체를 해결하는 것은 아니지만 알려진 공격을 차단하는 데 도움이 됩니다. 대안으로 인해 기능이 저하되는 경우는 아래 항목에 별도로 표시하였습니다.
  
-   **Exchange 2000 Server 및 IIS를 실행하는 서버에서 사용하는 모든 이벤트 싱크를 사용 안 함:**
  
    모든 이벤트 싱크를 사용하지 않도록 설정하면 이 취약점을 악용하려는 시도로부터 영향을 받는 시스템을 보호하는 데 도움이 됩니다. 모든 사용자 지정 이벤트 싱크 또는 타사 이벤트 싱크를 사용하지 않도록 설정하려면 다음 단계를 수행하십시오.
  
    1.  명령줄 명령 **cscript.exe smtpreg.vbs /enum**을 사용하여 등록된 이벤트 싱크를 모두 열거합니다. 이 명령의 출력을 검토하여 사용자 지정 또는 Microsoft가 아닌 타사 소프트웨어 이벤트 싱크 항목을 찾습니다.  
    2.  사용하지 않을 이벤트 싱크를 확인한 다음, 명령 프롬프트에서 다음 명령을 실행합니다.  
        **cscript.exe smtpreg.vbs /disable**
  
    **대안의 영향:** SMTP 서비스에서 이벤트 싱크를 만든 사용자 지정 또는 타사 응용 프로그램은 이벤트 싱크를 사용하도록 다시 설정할 때까지 제대로 작동하지 않습니다.
  
-   Exchange 2000 Server에서 **Cdoex.dll 파일, Cdosys.dll 파일 등록 취소 및 IIS 실행 서버에서 Cdosys.dll 파일 등록 취소**
  
    Cdoex.dll 파일 및 Cdosys.dll 파일을 등록 취소하면 영향을 받는 시스템에서 이 취약점이 악용되는 것을 막는 데 도움이 됩니다.
  
    Exchange 2000 Server용 Cdoex.dll 파일 및 Cdosys.dll 파일을 등록 취소하려면 다음 절차를 따르십시오.  
    **중요** 다음 순서대로 절차를 수행해야 합니다.
  
    1.  명령 프롬프트에서 다음 명령을 실행합니다.  
        **Regsvr32.exe “C:\\Program Files\\Common Files\\Microsoft Shared\\CDO\\cdoex.dll” /u**  
    2.  명령 프롬프트에서 다음 명령을 실행합니다.  
        **Regsvr32.exe %windir%\\system32\\cdosys.dll /u**
  
    IIS를 실행하는 서버에서 Cdosys.dll 파일을 사용하지 않도록 설정하려면 명령 프롬프트에 다음 명령을 입력합니다.
  
    1.  **Regsvr32.exe %windir%\\system32\\cdosys.dll /u**
  
    **대안의 영향:** COM 개체를 등록 취소하면 Cdosys.dll 파일 또는 Cdoex.dll 파일에 의존하는 프로그램이 중단됩니다.
  
#### Collaboration Data Objects 취약점 - CAN-2005-1987에 대한 FAQ:
  
**이 취약점을 악용하면 어느 작업까지 가능합니까?**  
이것은 원격 코드 실행 취약점입니다. 이 취약점을 악용한 공격자는 영향을 받는 시스템을 원격으로 완전히 제어할 수 있습니다. 이렇게 되면 공격자가 프로그램을 설치할 수 있을 뿐 아니라 데이터를 보거나 변경하거나 삭제할 수 있고 모든 사용자 권한이 있는 새 계정을 만들 수도 있습니다.
  
**취약점의 원인은 무엇입니까?**  
Collaboration Data Objects (CDO)의 검사되지 않은 버퍼가 원인입니다.
  
**Collaboration Data Objects란 무엇입니까?**  
Collaboration Data Objects (CDO)는 특히 인터넷 메일 메시지를 작성하거나 수정하는 프로그램을 간단히 제작할 수 있도록 설계된 구성 요소 개체 모델 (COM) 구성 요소입니다.
  
**공격자는 취약점을 악용하여 무엇을 할 수 있습니까?**  
취약점 악용에 성공한 공격자는 영향을 받는 시스템을 완전히 제어할 수 있습니다.
  
**누가 취약점을 악용할 수 있습니까?**  
Microsoft Windows 또는 Microsoft Exchange 2000 Server에서 특수하게 조작된 메시지를 영향을 받는 시스템에 전달할 수 있는 모든 익명의 사용자가 이 취약점을 악용할 수 있습니다.
  
**공격자는 이러한 취약점을 어떻게 악용합니까?**  
공격자는 영향을 받는 시스템에서 CDOSYS나 CDOEX에 의해 처리되는 특수하게 조작된 메시지를 만들어 취약점을 악용할 수 있습니다. 처리되면 SMTP를 실행하는 시스템을 제어하는 데 사용될 수 있습니다.
  
**취약점으로 인해 주로 공격 받을 위험이 있는 시스템은 무엇입니까?**  
CDOSYS 또는 CDOEX의 함수를 호출하는 응용 프로그램을 사용하여 메시지를 처리하는 경우 Microsoft Windows 및 Microsoft Exchange 2000 Server가 주로 위험합니다.
  
**업데이트는 어떤 기능을 합니까?**  
이 업데이트는 CDO가 메시지를 할당된 버퍼로 전달하기 전에 메시지 길이에 대한 유효성을 검사하도록 검사 방식을 수정함으로써 취약점을 제거합니다.
  
**이 보안 공지를 게시했을 때 이 취약점이 공개되었습니까?**  
아니요. Microsoft는 신뢰할 수 있는 정보 공개를 통해 이 취약점 관련 정보를 입수하였습니다. Microsoft는 이 보안 공지를 처음에 게시했을 때 이 취약점이 공개되었다는 정보를 확인하지 못했습니다.
  
**이 보안 공지를 게시했을 때 Microsoft는 이 취약점이 악용되었다는 보고를 받았습니까?**  
아니요. Microsoft는 이 취약점이 공개적으로 고객을 공격하는 데 사용되었다는 정보를 받지 못했으며, 이 보안 공지를 처음에 게시했을 때 개념 증명 코드(proof of concept code)의 예를 보지 못했습니다.

MBSA에 대한 자세한 내용은 [Microsoft Baseline Security Analyzer 웹 사이트](http://www.microsoft.com/korea/technet/security/tools/mbsahome.asp)를 참조하십시오.

-   **파일 버전 확인**

    **참고** Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색 결과** 창의 **검색 도우미**에서 **모든 파일 및 폴더**를 클릭합니다.
    3.  **전체 또는 일부 파일 이름 상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음, 해당 **속성**을 클릭합니다.  
        **참고** 설치된 프로그램 또는 운영 체제 버전에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 컴퓨터에 설치된 파일 버전을 확인합니다.  
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 또한 특정한 경우에는 파일 이름이 설치 중 변경될 수 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.
