Microsoft 보안 공지 MS05-050 - 긴급
===================================

DirectShow의 취약점으로 인한 원격 코드 실행 문제점 (904706)
-----------------------------------------------------------

게시된 날짜: 2005년 10월 12일 수요일 | 업데이트된 날짜: 2005년 12월 14일 수요일

**버전:** 1.3

#### 요약

**이내용을읽어야할사람:** Microsoft Windows를 사용하는 고객

**취약점으로인한영향:** 원격 코드 실행

**최대심각도:** 긴급

**권장사항:** 즉시 업데이트를 적용하십시오.

**보안업데이트대체:** 이 공지는 이전의 보안 업데이트를 대체합니다. 전체 목록을 보려면 이 공지의 자주 제기되는 질문 사항(FAQ) 항목을 참조하십시오.

**주의사항:** 고객이 이 보안 업데이트를 설치할 때 겪을 수 있는 현재까지 알려진 문제점은 [Microsoft 기술 자료 문서 905915](http://support.microsoft.com/kb/905915)에 나와 있습니다. 이 문서에는 이러한 문제점에 대한 권장 해결 방법도 나와 있습니다. 자세한 내용은 [Microsoft 기술 자료 문서 904706](http://support.microsoft.com/kb/904706)을 참조하십시오.

이 업데이트는 새로 발견되어 비공개적으로 보고된 취약점을 해결합니다. 이 공지에서 취약점은 "취약점 세부 사항" 항목에 설명되어 있습니다.

고객은 업데이트를 즉시 적용하는 것이 좋습니다.

이 보안 업데이트와 관련된 자주 제기되는 질문 사항  
-------------------------------------------------
  
 
**Microsoft가** **2005년** **12월** **13일이공지를다시공지한이유**  
Microsoft는 Windows 2000 서비스 팩 4, Windows XP 서비스 팩 1 및 Windows 2003에 대한 보안 공지가 “영향을 받는 소프트웨어" 섹션에 나열된 내용과 같이 개정되어 사용 가능함을 고객에게 알리기 위해 오늘 이 공지를 다시 공개했습니다. 자세한 내용은 "Microsoft가 2005년 11월 9일 이 공지를 업데이트한 이유"를 참조하십시오.
  
잘못된 버전의 DirectX 패키지가 시스템에 설치된 경우 설치가 실패하는 문제를 수정하기 위하여 업데이트가 개정되었습니다. 해당 Windows 버전에 적절한 DirectX 버전을 적용한 고객은 아무 조치를 취할 필요가 없습니다. WU, SUS, WSUS, MU, SMS를 사용하는 고객은 적절한 버전이 자동으로 감지되고 적용되기 때문에 아무 조치를 취하지 않아도 됩니다. 적절하지 않은 DirectX 패키지를 수동으로 설치한 고객은 시스템을 검사하고 적절한 업데이트를 재배포하여 정확한 DirectX 버전으로 업데이트하는 것이 좋습니다.
  
**Microsoft가** **2005년** **11월** **9일이공지를업데이트한이유**  
Microsoft는 DirectX 8 또는 DirectX 9가 설치된 컴퓨터에 "Windows 2000 서비스 팩 4에 설치된 DirectX 7.0용 보안 업데이트" 패키지를 설치할 때 설치는 아무 지시 없이 성공적으로 완료되지만 컴퓨터가 업데이트되지 않는 일부 사례를 확인했습니다. Microsoft는 DirectX 9가 설치된 컴퓨터에 "Windows XP 서비스 팩 1에 설치된 DirectX 8.1용 보안 업데이트” 또는 "Windows 2003에 설치된 DirectX 8.1용 보안 업데이트" 패키지를 설치할 때 설치는 아무 지시 없이 성공적으로 완료되지만 컴퓨터가 업데이트되지 않는 것도 확인했습니다. DirectX 8 또는 DirectX 9를 현재 실행하고 있는 사용자는 해당 DirectX 버전에 적절한 패키지를 적용할 때까지 MS05-050에서 다룬 문제점에 계속 취약할 수 있습니다. 자세한 내용은 FAQ에서 “고객이 이 보안 업데이트를 설치할 때 겪을 수 있는 현재 알려진 문제점은 무엇입니까?”를 참조하십시오.
  
**고객이이보안업데이트를설치할때겪을수있는현재알려진문제점은무엇입니까?**  
고객이 이 보안 업데이트를 설치할 때 겪을 수 있는 현재까지 알려진 문제점은 [Microsoft 기술 자료 문서 905915](http://support.microsoft.com/kb/905915)에 나와 있습니다. 이 문서에는 이러한 문제점에 대한 권장 해결 방법도 나와 있습니다. 자세한 내용은 [Microsoft 기술 자료 문서 904706](http://support.microsoft.com/kb/904706)을 참조하십시오.
  
-   [Microsoft 기술 자료 문서 909596](http://support.microsoft.com/kb/909596): Windows Server 2003, Windows XP 또는 Windows 2000을 실행하는 컴퓨터에 보안 공지 MS05-050과 연결된 DirectX 보안 업데이트 중 하나를 설치해도 컴퓨터가 업데이트되지 않았을 수 있음(909596) (MS05-050)  
-   [Microsoft 기술 자료 문서 910415](http://support.microsoft.com/kb/910415): Windows 2000 MUI(Multilingual User Interface)를 실행하는 Windows 2000 기반 컴퓨터에 보안 공지 MS05-050와 연결된 DirectX 보안 업데이트 중 하나를 설치해도 컴퓨터가 업데이트되지 않았을 수 있습니다(910415).
  
**컴퓨터에서** **Windows 2000** **서비스팩** **4를실행하고있습니다. DirectX 8.0, 8.0a, 8.1, 8.1a, 8.1b, 8.2** **또는** **9.0, 9.0a, 9.0b, 9.0c** **버전중하나가설치되어있으면,** **이문제에취약한상태입니까?**  
예. DirectX 7.0 보안 업데이트를 성공적으로 설치한 경우에도 DirectX 7.0 이상 DirectX 버전용 파일은 업데이트하지 않습니다.
  
**컴퓨터에서** **Windows XP** **서비스팩** **1을실행하고있습니다. DirectX 9.0, 9.0a, 9.0b, 9.0c** **버전중하나가설치되어있으면, DirectX 8.1** **업데이트를설치해도이문제에취약한상태입니까?**  
예. DirectX 8.1 보안 업데이트를 성공적으로 설치한 경우에도 DirectX 8.1 이상 DirectX 버전용 파일은 업데이트하지 않습니다.
  
**컴퓨터에서** **Windows 2003을실행하고있습니다. DirectX 9.0, 9.0a, 9.0b, 9.0c** **버전중하나가설치되어있으면, DirectX 8.1** **업데이트를설치해도이문제에취약한상태입니까?**  
예. DirectX 8.1 보안 업데이트를 성공적으로 설치한 경우에도 DirectX 8.1 이상 DirectX 버전용 파일은 업데이트하지 않습니다.
  
**컴퓨터에서** **Windows 2000** **서비스팩** **4, Windows XP** **서비스팩** **1** **또는** **Windows 2003을실행하고있습니다.** **정확한보안업데이트를설치했는지어떻게알수있습니까?**  
정확한 보안 업데이트를 설치했는지 확인하려면 MBSA 2.0, Enterprise Update Scanning Tool(EST), SUS, WSUS 또는 SMS를 실행하십시오. Windows Update 웹 사이트를 방문해도 정확한 보안 업데이트를 설치했는지 확인할 수 있습니다. 파일 확인에 대한 자세한 내용은 “DirectX 독립 실행형” 섹션에서 “Windows 2000에 설치된 DirectX 8 및 DirectX 9 모든 버전”, “Windows XP에 설치된 DirectX 9 모든 버전” 또는 “Windows 2003에 설치된 DirectX 9 모든 버전” 항목을 참조하십시오.

**어떤업데이트가시스템에적용되는지어떻게확인할수있습니까?**  
  
이 업데이트의 시작 부분에 나열된 **다운로드**는 두 가지 범주로 나뉩니다.
  
1.  영향을 받는 소프트웨어  
2.  영향을 받는 구성 요소
  
"영향을 받는 소프트웨어"의 운영 체제 옆에 나열된 다운로드는 보다 최신 버전의 DirectX로 업그레이드되지 않은 시스템에 해당됩니다. 시스템의 DirectX가 업그레이드된 경우, 사용자는 "영향을 받는 구성 요소" 항목에서 해당 시스템의 현재 DirectX 버전과 일치하는 다운로드를 설치해야 합니다.
  
**시스템에서업데이트된버전의** **DirectX를실행하는지어떻게확인합니까?**

#### DirectShow 취약점 - CAN-2005-2128에 대한 FAQ:

**이취약점을악용하면어느작업까지가능합니까?**  
이것은 원격 코드 실행 취약점입니다. 이 취약점을 악용한 공격자는 영향을 받는 시스템을 원격으로 완전히 제어할 수 있습니다. 공격자가 프로그램을 설치할 수 있을 뿐 아니라 데이터를 보거나 변경하거나 삭제할 수 있고 모든 사용자 권한이 있는 새 계정을 만들 수도 있습니다.

**취약점의원인은무엇입니까?**  
DirectShow의 검사되지 않는 버퍼가 원인입니다.

**DirectShow란무엇입니까?**  
Microsoft DirectShow는 Microsoft Windows 운영 체제에서 미디어 스트리밍에 사용됩니다. DirectShow는 고화질 멀티미디어 스트림의 캡처와 재생에 사용됩니다. 사용 가능한 비디오 및 오디오 가속 하드웨어를 자동으로 검색하고 사용하지만, 가속 하드웨어가 없는 시스템도 지원합니다. 다른 DirectX 기술과도 통합됩니다.
DVD 플레이어, 비디오 편집 응용 프로그램, AVI - ASF 변환기, MP3 플레이어 및 디지털 비디오 캡처 응용 프로그램 등이 DirectShow를 사용합니다.

**공격자는취약점을악용하여무엇을할수있습니까?**  
취약점 악용에 성공한 공격자는 영향을 받는 시스템을 완전히 제어할 수 있습니다.

**누가취약점을악용할수있습니까?**  
Windows 운영 체제에서 익명 사용자가 특수하게 조작한 AVI 파일을 영향을 받는 시스템에 전달하여 이 취약점 악용을 시도할 수 있습니다.

전자 메일을 통한 공격의 경우, 공격자는 특수하게 조작한 AVI 파일을 사용자에게 보내고 열도록 유도하여 이 취약점 악용을 시도할 수 있습니다.
웹을 통한 공격의 경우, 공격자는 호스팅하는 웹 사이트에 이 취약점을 악용하는 웹 페이지를 포함해야 합니다. 공격자는 강제로 사용자가 악의적인 웹 사이트를 방문하도록 할 수 없습니다. 대신에 공격자는 사용자가 공격자의 사이트로 연결되는 링크를 누르게 하여 공격자의 웹 사이트를 방문하도록 유인해야 합니다. 배너 광고나 다른 방법을 통해 영향 받는 시스템에 웹 콘텐츠를 보내는 방법으로 악의적인 웹 콘텐츠를 표시할 수도 있습니다.

**취약점으로인해주로공격받을위험이있는시스템은무엇입니까?**  
워크스테이션과 터미널 서버가 주로 위험합니다. 서버는 충분한 관리 권한이 없는 사용자가 서버에 로그온하고 프로그램을 실행할 수 있는 경우에 더 위험합니다. 가장 좋은 방법은 이러한 기능을 허용하지 않는 것입니다.

**업데이트는어떤기능을합니까?**  
이 업데이트는 DirectShow가 메시지를 할당된 버퍼로 전달하기 전에 메시지 길이에 대한 유효성을 검사하는 방식을 수정함으로써 취약점을 제거합니다.

**이보안공지를게시했을때이취약점이공개되었습니까?**  
아니요. Microsoft는 신뢰할 수 있는 정보 공개를 통해 이 취약점 관련 정보를 입수하였습니다. Microsoft는 이 보안 공지를 처음에 게시했을 때 이 취약점이 공개되었다는 정보를 확인하지 못했습니다.

**이보안공지를게시했을때** **Microsoft는이취약점이악용되었다는보고를받았습니까?**

**적용된업데이트확인**

-   **Microsoft Baseline Security Analyzer**

    MBSA(Microsoft Baseline Security Analyzer) 도구를 사용하면 영향을 받는 시스템에 보안 업데이트가 적용되었는지 확인할 수 있습니다. 관리자는 MBSA를 사용하여 로컬 및 원격 시스템에서 누락된 보안 업데이트와 일반적인 보안 구성 오류를 검색할 수 있습니다. MBSA에 대한 자세한 내용은 [Microsoft Baseline Security Analyzer 웹 사이트](http://www.microsoft.com/korea/technet/security/tools/mbsahome.asp)를 참조하십시오.

-   **파일버전확인**

    **참고** Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색결과** 창의 **검색도우미**에서 **모든파일및폴더**를 클릭합니다.
    3.  **전체또는일부파일이름상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음, 해당 **속성**을 클릭합니다.  
        **참고** 설치된 프로그램 또는 운영 체제 버전에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 컴퓨터에 설치된 파일 버전을 확인합니다.  
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 또한 특정한 경우에는 파일 이름이 설치 중 변경될 수 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

-   **레지스트리키확인**

    다음 레지스트리 키를 검토하여 보안 업데이트로 설치된 파일을 확인할 수도 있습니다.

    Windows XP Home Edition 서비스 팩 1, Windows XP Professional 서비스 팩 1, Windows XP Tablet PC Edition, Windows XP Media Center Edition, Windows XP Home Edition 서비스 팩 2, Windows XP Professional 서비스 팩 2, Windows XP Tablet PC Edition 2005 및 Windows XP Media Center Edition 2005:

MBSA에 대한 자세한 내용은 [Microsoft Baseline Security Analyzer 웹 사이트](http://www.microsoft.com/korea/technet/security/tools/mbsahome.asp)를 참조하십시오. MUI를 실행하는 Windows 2000 SP4 사용자가 파일 버전을 정확하게 확인하는 방법은 [기술 자료 문서 263212](http://support.microsoft.com/?kbid=263212)를 참조하십시오.

-   **파일버전확인**

    **참고** Microsoft Windows에는 여러 버전이 있으므로 사용자의 컴퓨터에서 다음 단계가 다르게 적용될 수 있습니다. 이러한 경우 제품 설명서를 참조하여 작업 단계를 완료하십시오. MUI를 실행하는 Windows 2000 SP4 사용자가 파일 버전을 정확하게 확인하는 방법은 [기술 자료 문서 263212](http://support.microsoft.com/?kbid=263212)를 참조하십시오.

    1.  **시작**, **검색**을 차례로 클릭합니다.
    2.  **검색결과** 창의 **검색도우미**에서 **모든파일및폴더**를 클릭합니다.
    3.  **전체또는일부파일이름상자**에 적절한 파일 정보 표의 파일 이름을 입력한 다음 **검색**을 클릭합니다.
    4.  파일 목록에서 해당 파일 정보 표의 파일 이름을 마우스 오른쪽 단추로 클릭한 다음, 해당 **속성**을 클릭합니다.  
        **참고** 설치된 프로그램 또는 운영 체제 버전에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 컴퓨터에 설치된 파일 버전을 확인합니다.  
        **참고** 파일 버전 이외의 특성은 설치 중 변경될 수 있습니다. 다른 파일 특성과 파일 정보 표에 있는 정보를 비교하는 것은 지원되는 업데이트 적용 확인 방법이 아닙니다. 또한 특정한 경우에는 파일 이름이 설치 중 변경될 수 있습니다. 파일 또는 버전 정보가 없으면 사용 가능한 다른 방법 중 하나를 사용하여 업데이트 설치를 확인하십시오.

    다음 레지스트리 키를 검토하여 보안 업데이트가 설치한 파일을 확인할 수도 있습니다.

    1.  **시작**, **검색**을 차례로 클릭합니다.

        **참고** 설치된 프로그램 또는 운영 체제 버전에 따라 파일 정보 표에 나열된 파일 중 일부가 설치되어 있지 않을 수도 있습니다.
    5.  **버전** 탭의 버전을 해당 파일 정보 표에 있는 버전과 비교하여 사용자 컴퓨터에 설치된 파일 버전을 확인합니다.
